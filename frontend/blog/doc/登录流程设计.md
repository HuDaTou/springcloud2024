## 三、Token 管理设计

### 3.1 存储格式

Token 以 JSON 字符串形式存储，包含 token 值和过期时间戳：

```json
{"token": "eyJ...", "expire": 1719600000000}
```

### 3.2 存储策略

| 场景 | 存储位置 | 生命周期 |
|------|---------|---------|
| 用户勾选"记住我" | `localStorage` | 持久化，除非手动清除或过期 |
| 用户未勾选"记住我" | `sessionStorage` | 关闭浏览器即失效 |

### 3.3 过期校验

每次 `GET_TOKEN()` 读取时都会检查 `expire < Date.now()`：
- 未过期 → 返回 token 字符串
- 已过期 → 自动清除存储，弹出提示"登录状态已过期，请重新登录"

### 3.4 操作函数 (src/utils/auth.ts)

| 函数 | 作用 |
|---|---|
| `GET_TOKEN()` | 读取并校验过期，返回 token 或 null |
| `SET_TOKEN(token, expire, remember)` | 根据 remember 选择存储位置，同时写入 Pinia store |
| `REMOVE_TOKEN()` | 同时清除 localStorage 和 sessionStorage 中的 Token |

## 四、登录方式

### 4.1 方式一：用户名密码登录

```
用户 → 访问 /welcome/login
     → 填写 username + password + remember
     → POST /cloud-auth/auth/login
     → 后端返回 {accessToken, expire}
     → SET_TOKEN(token, expire, remember)
     → router.push('/')
     → userStore.getInfo() → GET /user/auth/info
```

### 4.2 方式二：第三方 OAuth 登录（Gitee / GitHub）

```
用户 → 点击 Gitee/GitHub 图标
     → 跳转后端 /oauth/gitee/render 或 /oauth/github/render
     → 在第三方平台完成授权
     → 回调到前端，URL 携带: ?access_token=xxx&login_type=gitee&user_name=xxx
     → Header 组件 onMounted 检测到 URL 参数
     → POST /user/login（自定义 Header: Login-Type, Access-Token）
     → 后端返回 {accessToken, expire}
     → router.replace({query: {}}) 清除 URL 参数
     → SET_TOKEN() → 跳转首页 → getInfo()
```

---

## 五、退出登录

```
用户 → 点击 Header 中的"退出登录"
     → POST /user/logout
     → REMOVE_TOKEN() 清除 localStorage + sessionStorage
     → userStore.userInfo = undefined
     → router.push('/')
```

## 六、Axios 请求拦截器

所有请求自动携带 Token：

```ts
// 请求拦截器
config.headers['X-Client-Type'] = 'Frontend'
if (GET_TOKEN()) {
    config.headers['Authorization'] = 'Bearer ' + GET_TOKEN()
}

// 响应拦截器
// code === 1012 → 提示"账号已被封禁"
```

## 七、路由守卫

`src/router/index.ts` 中的全局前置守卫：

```ts
router.beforeEach((to, from, next) => {
    const isLogin = GET_TOKEN()
    // 已登录用户访问登录/注册/重置 → 重定向到首页
    if (to.name?.startsWith('welcome-') && isLogin) {
        next('/')
    } else {
        next()
    }
})
```

设计特点：只阻止已登录用户重复登录，所有页面均可无登录浏览（无强制登录保护）。

## 八、Pinia 用户 Store

`src/store/modules/user.ts`：

```ts
useUserStore:
  state:
    token:   从 GET_TOKEN() 恢复
    userInfo: shallowRef<UserInfo>(昵称、头像、角色、权限等)
  actions:
    getInfo(): 调用 getUserInfo() API 拉取完整用户信息
```

`token` 在 store 创建时从 storage 恢复，`userInfo` 在登录成功后通过 `getInfo()` 拉取。

## 九、注册流程

```
用户 → 访问 /welcome/register
     → 填写 username + password + 重复密码 + email
     → 点击"获取验证码" → POST /cloud-auth/auth/send-code (type=register)
     → 60秒冷却后可以重新发送
     → 点击"立即注册" → POST /cloud-auth/auth/register
     → 注册成功 → router.push('/login')
```

---

## 十、重置密码流程

两步流程（el-steps 展示）：

```
第一步：输入邮箱 + 验证码
       → 点击"获取验证码" → POST /cloud-auth/auth/send-code (type=reset)
       → 下一步 → POST /user/reset-confirm 验证

第二步：输入新密码 + 重复密码
       → 提交 → POST /user/reset-password
       → 成功 → "密码重置成功，请重新登录" → push /login
```
