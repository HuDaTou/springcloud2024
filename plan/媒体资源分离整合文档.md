## 一、项目概述

### 1.1 目标
将 `cloud-web` 服务中的文件上传操作迁移到使用 Feign 调用 `cloud-Media-Asset-Processing` 服务，实现媒体资源的统一管理和微服务架构的职责分离。

### 1.2 涉及服务
- **源服务**: `cloud-web` - 当前直接处理文件上传
- **目标服务**: `cloud-Media-Asset-Processing` - 专门处理媒体资源的服务
- **Feign 客户端**: `cloud-api` - 提供 Feign 接口定义

### 1.3 迁移范围
- 文章相关：封面、图片上传
- Banner 管理：图片上传
- 照片管理：照片上传
- 视频管理：视频、封面上传

### 1.4 核心价值
| 维度 | 价值 |
|------|------|
| **架构优化** | 职责清晰，易于维护 |
| **技术升级** | 使用预签名URL，更安全高效 |
| **性能提升** | 支持大文件分片上传和并发处理 |
| **安全增强** | 预签名URL机制更安全 |

## 二、当前架构分析

### 2.1 当前实现方式
`cloud-web` 服务直接使用 MinIO SDK 进行文件上传，存在以下问题：
- ❌ 职责不清晰：业务服务直接处理文件存储

- ❌ 维护困难：文件存储逻辑分散在多个服务中

└─────────────────────────────────────────────────────────────┘

### 3.2 预签名URL上传流程

```
客户端 → cloud-web → Feign → cloud-Media → MinIO
   ↓         ↓         ↓         ↓         ↓
请求上传  获取URL   返回URL   生成URL   验证URL
   ↓         ↓         ↓         ↓         ↓
上传文件  使用URL   上传到MinIO  验证文件  存储文件
   ↓         ↓         ↓         ↓         ↓
完成上传  通知完成  更新状态  记录资产  返回结果
```

### 3.3 分片上传流程（大文件）

```
1. 初始化分片上传 → 获取 uploadId 和分片URL列表
2. 并行上传分片   → 每个分片独立上传
3. 完成分片上传   → 合并所有分片
```

#### 步骤 2：迁移 uploadArticleCover()

**原实现**：
```java
@Override
public ResultData<String> uploadArticleCover(MultipartFile articleCover) {
    try {
        String articleCoverUrl = fileUploadUtils.uploadImage(UploadEnum.ARTICLE_COVER, articleCover);
        return ResultData.success(articleCoverUrl);
    } catch (FileUploadException e) {
        return ResultData.failure(e.getMessage());
    }
}
```

**新实现**：
```java
@Override
public ResultData<String> uploadArticleCover(MultipartFile articleCover) {
    try {
        PresignedUploadRequest request = new PresignedUploadRequest(
            articleCover.getOriginalFilename(),
            articleCover.getSize(),
            articleCover.getContentType(),
            null,
            "ARTICLE_COVER"
        );

        Long userId = SecurityUtils.getUserId();
        ResultData<Map<String, Object>> presignResult = mediaClient.getPresignedUploadUrl(userId, request);

        if (!presignResult.isSuccess()) {
            return ResultData.failure("获取上传URL失败");
        }

        Map<String, Object> data = presignResult.getData();
        String presignedUrl = (String) data.get("presignedUrl");
        String objectName = (String) data.get("objectName");

        minioPresignedUploadHelper.uploadWithPresignedUrl(presignedUrl, articleCover);

        ResultData<Long> completeResult = mediaClient.completeUpload(objectName);
        if (!completeResult.isSuccess()) {
            return ResultData.failure("完成上传失败");
        }

        ResultData<String> urlResult = mediaClient.getFileUrl(objectName);
        return urlResult;
    } catch (Exception e) {
        log.error("文章封面上传失败", e);
        return ResultData.failure("文章封面上传失败: " + e.getMessage());
    }
}
```

```java
@Override
public Map<String, Object> uploadVideo(MultipartFile videoFile) {
    try {
        validateVideoFile(videoFile);

        InitiateMultipartUploadRequest request = new InitiateMultipartUploadRequest(
            videoFile.getOriginalFilename(), videoFile.getSize(),
            videoFile.getContentType(), null, "VIDEO");

        Long userId = SecurityUtils.getUserId();
        ResultData<Map<String, Object>> initiateResult = mediaClient.initiateMultipartUpload(userId, request);

        if (!initiateResult.isSuccess()) {
            throw new RuntimeException("初始化分片上传失败");
        }

        Map<String, Object> data = initiateResult.getData();
        String uploadId = (String) data.get("uploadId");
        Map<String, String> partUrls = (Map<String, String>) data.get("partUrls");

        minioPresignedUploadHelper.uploadParts(partUrls, videoFile);
        mediaClient.completeMultipartUpload(uploadId);

        String objectName = (String) data.get("objectName");
        ResultData<String> urlResult = mediaClient.getFileUrl(objectName);

        return Map.of("video", urlResult.getData(),
            "videoSize", convertVideoSize(videoFile.getSize()),
            "videoType", videoFile.getContentType());
    } catch (Exception e) {
        log.error("视频上传失败", e);
        throw new RuntimeException("视频上传失败", e);
    }
}
```

## 五、辅助工具类设计

### 5.1 MinIO 预签名URL上传工具

```java
package com.overthinker.cloud.web.utils;

import okhttp3.*;
import org.springframework.stereotype.Component;
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Component
public class MinioPresignedUploadHelper {

    private final OkHttpClient client = new OkHttpClient();

    public void uploadWithPresignedUrl(String presignedUrl, MultipartFile file) {
        try {
            RequestBody requestBody = RequestBody.create(
                file.getBytes(), MediaType.parse(file.getContentType()));

            Request request = new Request.Builder()
                .url(presignedUrl)
                .put(requestBody)
                .build();

            Response response = client.newCall(request).execute();
            if (!response.isSuccessful()) {
                throw new RuntimeException("上传失败: HTTP " + response.code());
            }

            log.info("文件上传成功: {}", file.getOriginalFilename());
        } catch (Exception e) {
            log.error("使用预签名URL上传失败", e);
            throw new RuntimeException("上传失败: " + e.getMessage(), e);
        }
    }

    public void uploadParts(Map<String, String> partUrls, MultipartFile file) {
        // 实现分片上传逻辑
    }
}
```

### 5.2 URL解析工具

```java
package com.overthinker.cloud.web.utils;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class MediaUrlUtils {

    @Value("${minio.bucketName}")
    private String bucketName;

    public String extractObjectName(String fullUrl) {
        int bucketIndex = fullUrl.indexOf(bucketName);
        if (bucketIndex == -1) {
            throw new IllegalArgumentException("URL中未找到bucketName");
        }
        return fullUrl.substring(bucketIndex + bucketName.length() + 1);
    }

    public String extractFileName(String url) {
        int lastSlash = url.lastIndexOf('/');
        return lastSlash == -1 ? url : url.substring(lastSlash + 1);
    }
}
```

## 七、错误处理

### 7.1 统一异常处理

```java
@Component
public class MediaUploadExceptionHandler {

    public ResultData<String> handleUploadException(Exception e, String operation) {
        log.error("{}失败", operation, e);

        if (e instanceof FeignException) {
            return ResultData.failure("媒体服务调用失败: " + e.getMessage());
        } else if (e instanceof RuntimeException) {
            return ResultData.failure("上传失败: " + e.getMessage());
        } else {
            return ResultData.failure("系统错误");
        }
    }
}
```

## 十三、常见问题FAQ

### Q1: 为什么使用预签名URL而不是直接上传？
**A**: 预签名URL上传有以下优势：
- 安全性：URL有时效限制，避免长期暴露
- 解耦：cloud-web 不需要 MinIO SDK
- 灵活性：可以在客户端直接上传，减少服务器负载

### Q2: 大文件上传如何处理？
**A**: 对于 >5MB 的文件：
- 使用分片上传
- 每个分片独立上传，失败可重试
- 支持断点续传

### Q3: 如何处理上传失败？
**A**: 实现三层保障：
1. Feign调用重试机制（3次）
2. 本地临时存储作为备份
3. 定时任务清理未完成的资产记录

### Q4: 性能会下降吗？
**A**: 合理优化后性能不会下降：
- 使用异步上传减少等待时间
- 预签名URL上传直接到MinIO，无中间转发
- 分片上传支持并行处理
