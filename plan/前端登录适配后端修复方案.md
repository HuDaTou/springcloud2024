# 前端登录适配后端修复方案

## 概述

前端博客项目（`frontend/blog`）的登录功能当前无法正常使用。经对比 [前端登录流程设计](../frontend/blog/doc/登录流程设计.md) 与 [后端登录逻辑设计](../cloud-auth/docs/登录逻辑设计.md) 及实际代码，发现 **7 项前后端不一致问题**，其中 **1 项为致命 Bug（登录成功被判定为失败）**。

> **暂不涉第三方登录（Gitee/GitHub）**，相关问题待后端 OAuth 实现完成后再对接。本次仅处理账号密码登录。

---

## 网关架构说明

```
浏览器 → Gateway (9527) → cloud-auth (9123)
```

- 网关路由前缀：`/cloud-auth/**` → 转发至 `cloud-auth` 服务（自动 strip 前缀）
- 后端直接路径：`/auth/login` → 网关路径：`/cloud-auth/auth/login`

---

## 问题清单与修复方案

### 问题 1（致命 Bug）：登录响应 code 类型不匹配

| 项 | 值 |
|---|---|
| **严重程度** | 致命 — 登录永远失败 |
| **后端返回** | `{ "code": "200", "msg": "登录成功", "data": {...} }` |
| **前端判断** | `res.code === 200` |
| **问题** | `ResultData.code` 字段类型是 `String`，后端返回 `"200"`（字符串），前端用 `===` 与数字 `200` 比较，严格相等永远为 `false` |

**修复文件**: `frontend/blog/src/views/Welcome/Login/index.vue`（第 41 行）

**修复前**:
```ts
if (res.code === 200) {
```

**修复后**:
```ts
if (res.code === '200') {
```

---

### 问题 2：发送验证码接口路径错误

| 项 | 值 |
|---|---|
| **严重程度** | 高 — 注册/重置密码无法获取验证码 |
| **前端调用** | `POST /cloud-auth/auth/send-code` |
| **后端实际** | `POST /email/send-code`（EmailController，`@RequestMapping("/email")`） |
| **问题** | `SecurityConfig` 白名单中写了 `/auth/send-code`，但无 Controller 实现；实际实现在 `EmailController` 的 `/email/send-code` |

**修复文件**: `frontend/blog/src/apis/email/index.ts`

**修复前**:
```ts
export const sendEmail = (data: SendEmailDTO) =>
  http.post('/cloud-auth/auth/send-code', data)
```

**修复后**:
```ts
export const sendEmail = (data: SendEmailDTO) =>
  http.post('/cloud-auth/email/send-code', data)
```

> **后端需同步修复**: `SecurityConfig.java` 第 74 行白名单 `/auth/send-code` → `/email/send-code`

---

### 问题 3：退出登录接口路径错误

| 项 | 值 |
|---|---|
| **严重程度** | 高 — 退出功能不可用 |
| **前端调用** | `POST /user/logout` |
| **后端实际** | `POST /auth/logout`（SecurityConfig.logoutUrl） |
| **问题** | `/user/logout` 路径不存在，`SecurityConfig` 中配置的登出路径是 `/auth/logout` |

**修复文件**: `frontend/blog/src/apis/user/index.ts`（第 81 行）

**修复前**:
```ts
export const logout = () => http.post('/user/logout')
```

**修复后**:
```ts
export const logout = () => http.post('/cloud-auth/auth/logout')
```

> 注：`/user/logout` 不经过网关的 `/cloud-auth/**` 路由规则，无法到达 `cloud-auth` 服务。

```ts
export const getUserInfo = () => http.get('/user/auth/info')
```

**修复后**:
```ts
export const getUserInfo = () => http.get('/cloud-auth/user/auth/info')
```

### 问题 5：重置密码接口不存在

| 项 | 值 |
|---|---|
| **严重程度** | 高 — 重置密码功能不可用 |
| **前端调用** | `POST /user/reset-confirm`、`POST /user/reset-password` |
| **后端实际** | `AuthService` 中已实现 `resetConfirm()` 和 `resetPassword()` 方法，但**无 Controller 暴露** |

**修复方案（需要同时修改前后端）**:

**后端** — 在 `AuthController.java` 中添加两个端点：
```java
@PostMapping("/reset-confirm")
public ResultData<Void> resetConfirm(@RequestBody @Valid UserResetConfirmDTO dto) {
    return authService.resetConfirm(dto);
}

@PostMapping("/reset-password")
public ResultData<Void> resetPassword(@RequestBody @Valid UserResetPasswordDTO dto) {
    return authService.resetPassword(dto);
}
```

**前端** — 修改 `frontend/blog/src/apis/user/index.ts`：
```ts
// 修复前
export const resetPasswordStepOne = (data) => http.post('/user/reset-confirm', data)
export const resetPasswordStepTwo = (data) => http.post('/user/reset-password', data)

// 修复后
export const resetPasswordStepOne = (data) => http.post('/cloud-auth/auth/reset-confirm', data)
export const resetPasswordStepTwo = (data) => http.post('/cloud-auth/auth/reset-password', data)
```

### 问题 6：用户设置相关接口缺少网关前缀

| 项 | 值 |
|---|---|
| **严重程度** | 中 — 用户设置功能不可用 |
| **涉及接口** | `/user/auth/update`、`/user/auth/update/email`、`/user/auth/third/update/email` |
| **后端路径** | 对应 `UserController` 中的接口 |

**修复文件**: `frontend/blog/src/apis/user/index.ts`

```ts
// 修复前
export const updateUserAccount = (data) => http.post('/user/auth/update', data)
export const updateEmail = (data) => http.post('/user/auth/update/email', data)
export const updateThirdEmail = (data) => http.post('/user/auth/third/update/email', data)

// 修复后
export const updateUserAccount = (data) => http.post('/cloud-auth/user/auth/update', data)
export const updateEmail = (data) => http.post('/cloud-auth/user/auth/update/email', data)
export const updateThirdEmail = (data) => http.post('/cloud-auth/user/auth/third/update/email', data)
```

---

### 问题 7：全局 `res.code` 类型比较问题

| 项 | 值 |
|---|---|
| **严重程度** | 中 — 可能影响其他接口的判断 |
| **问题** | `ResultData.code` 在后端定义为 `String`，但前端各地可能用数字比较 |

**需排查文件**:
- `frontend/blog/src/views/Welcome/Register/index.vue`
- `frontend/blog/src/views/Welcome/Reset/index.vue`
- `frontend/blog/src/views/Setting/index.vue`
- 所有调用 API 后进行 `res.code === 200` 判断的组件

**修复**: 全局替换 `res.code === 200` → `res.code === '200'` 或改用类型安全比较 `String(res.code) === '200'`。
