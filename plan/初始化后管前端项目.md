### 具体实现方案
#### 🎨 阶段 1：删除前端的git初始化后台

前端项目 Vue Vben Admin 已从独立仓库物理合并到后端大仓库中，以下是具体操作步骤：

##### 步骤 1：删除前端 git 仓库
- 删除 `frontend/vue-vben-admin` 目录下的 `.git` 文件夹
- 使前端代码作为普通子目录合并到 `springcloud2024` 大仓库中
- 不提交代码，保持本地状态

##### 步骤 2：复制 web-antd 目录
- 复制 `frontend/vue-vben-admin/apps/web-antd` 目录
- 目标目录：`frontend/vue-vben-admin/apps/web-antd-bakend`

##### 步骤 3：修改 package.json name 字段
- 文件路径：`frontend/vue-vben-admin/apps/web-antd-bakend/package.json`
- 修改 `name` 字段为 `@vben/web-antd-bakend`
- 说明：Pnpm 识别项目依据的是 package.json 中的 name 字段，而非文件夹名。若不改，Pnpm 会发现两个同名项目，直接报错。

##### 步骤 4：安装 pnpm 依赖
- 在项目根目录 `frontend/vue-vben-admin` 下执行 `pnpm install`
- 等待所有依赖安装完成
- 确保安装过程中无报错

##### 步骤 5：配置 Vite 代理地址
- 文件路径：`frontend/vue-vben-admin/apps/web-antd-bakend/vite.config.ts`
- 修改代理 `target` 为后端网关地址：`http://localhost:9527`
- 作用：打通前后端，前端请求通过网关转发到后端微服务

##### 步骤 6：彻底关闭 Mock（确保请求走真实后端）

###### 6.1 关闭 Nitro Mock 服务
- 文件路径：`frontend/vue-vben-admin/apps/web-antd-bakend/.env.development`
- 确保 `VITE_NITRO_MOCK=false`
- 说明：此环境变量控制是否开启 Nitro Mock 服务

###### 6.2 修改 Vite 代理配置指向真实网关
- 文件路径：`frontend/vue-vben-admin/apps/web-antd-bakend/vite.config.ts`
- 将 `target` 从 `http://localhost:5320/api` 改为 `http://localhost:9527`
- 去掉代理 URL 中的 `/api` 后缀，直接指向网关地址
- 修改 `rewrite` 规则确保路径正确
