# 博客前端与后端连接配置计划

## 一、当前架构分析

### 1.1 网关配置现状

根据 [cloud-gatway9527/README.md](file:///root/code/java/gitcode/springcloud2024/cloud-gatway9527/README.md)：

**网关特性：**
- 只使用 **Consul 动态路由**（服务发现自动路由），**未配置静态路由**
- 路由格式：`http://网关:9527/{服务名}/{路径}`
- 示例：`/cloud-auth/auth/login` → 转发到 `cloud-auth` 服务

**当前白名单（需修改）：**
- `/actuator/**` ✅ 已放行
- `/cloud-auth/**` ✅ 已放行
- ❌ 其他所有请求需要 JWT Token

**问题：**
- 公开接口（文章列表、轮播图等）需要放行，但网关没有配置
- 前端使用 `/api` 前缀代理，但网关不支持 `/api` 路径

### 1.2 后端微服务端口

| 服务名称 | 端口 | Consul 注册名 |
|---------|------|-------------|
| cloud-gatway9527 | 9527 | cloud-gateway |
| cloud-auth | 9123 | cloud-auth |
| cloud-web | 8000 | cloud-web |

## 二、修改计划

### 阶段 1：修改网关白名单（核心步骤）

**目标**：将公开接口添加到白名单，无需 JWT Token 即可访问

**文件**：`cloud-gatway9527/src/main/java/com/overthinker/cloud/gateway/config/SecurityConfig.java`

**当前代码**：
```java
.pathMatchers("/actuator/**", "/cloud-auth/**").permitAll()
```

**修改为**：
```java
.pathMatchers(
    "/actuator/**",                    // 健康检查
    "/cloud-auth/**",                  // 认证服务（登录/注册）
    "/article/**",                    // 文章（公开）
    "/comment/**",                     // 评论（公开）
    "/banners/**",                     // 轮播图（公开）
    "/websiteInfo/**",                 // 网站信息（公开）
    "/link/**",                        // 友链（公开）
    "/photo/**",                       // 相册（公开）
    "/video/**",                       // 视频（公开）
    "/leaveWord/list"                  // 留言板列表（公开）
).permitAll()
```

**说明**：
- 此修改是**必须的**，否则未登录用户无法访问首页文章等公开内容
- `/auth/*` 退出登录不需要放行（需要 Token 才能退出）
- `/user/*`、`/favorite/*`、`/like/*` 不放行（需要登录才能访问）

---

### 阶段 2：前端环境变量配置

**目标**：确保前端正确配置网关地址

**创建文件**：`frontend/blog/.env.development`

```env
# 开发环境配置
VITE_SERVE=http://localhost:9527
VITE_APP_BASE_API=/api
VITE_FRONTEND_URL=http://localhost:5199
```

**说明**：
- `VITE_SERVE` 是 Vite 代理的目标地址（网关）
- `VITE_APP_BASE_API=/api` 让 axios 请求带上 `/api` 前缀

---

### 阶段 3：前端 Vite 代理配置

**目标**：将 `/api` 前缀的请求转发到网关

**文件**：`frontend/blog/vite.config.ts`

**当前配置**：
```typescript
proxy: {
    '/api': {
        target: `${loadEnv(mode, process.cwd()).VITE_SERVE}`,
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')  // ❌ 去掉了 /api
    },
}
```

**问题分析**：
- 当前配置会去掉 `/api` 前缀，导致请求变成 `/cloud-auth/auth/login`
- Consul 动态路由**不需要**去掉 `/api`，需要保留

**修改为**：
```typescript
proxy: {
    '/api': {
        target: `${loadEnv(mode, process.cwd()).VITE_SERVE}`,
        changeOrigin: true,
        // rewrite: (path) => path  // 保持 /api 前缀（Consul 动态路由需要）
    },
}
```

**路由流程**：
```
前端: /api/cloud-auth/auth/login
    ↓ Vite 代理
网关: /api/cloud-auth/auth/login
    ↓ Consul 动态路由（提取服务名）
cloud-auth:9123: /auth/login
```

### 阶段 4：前端硬编码 URL 修复

**目标**：统一前端代码中的 OAuth 登录路径

## 四、请求流程图

### 公开接口（无需登录）

```
浏览器 → 前端 → Vite代理(/api) → 网关 → cloud-web → 返回
         ↑                              ↑
      无需 Token                    白名单放行 ✅
```

### 需要登录的接口

```
浏览器 → 前端 → Vite代理(/api) → 网关 → cloud-auth(验证Token) → cloud-web → 返回
         ↑                              ↑                    ↑
      需携带 Token                  验证 JWT            业务处理
```
