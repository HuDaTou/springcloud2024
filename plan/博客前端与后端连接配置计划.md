### 1.3 当前前端 API 调用模式分析

#### 方式一：通过 Vite 代理转发（开发环境）

```
vite.config.ts 配置：
  proxy: {
    '/api': {
      target: `${loadEnv(mode, process.cwd()).VITE_SERVE}`,
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }

实际流程：
前端请求: /api/cloud-auth/auth/login
         ↓ Vite 代理重写
目标地址: {VITE_SERVE}/cloud-auth/auth/login
```

#### 方式二：直接使用 baseURL（axios 实例）

```typescript
// http.ts 第 13 行
const http: AxiosInstance = axios.create({
    baseURL: import.meta.env.VITE_APP_BASE_API ?? '/',
    ...
})

如果 VITE_APP_BASE_API = '/api'，那么：
  /cloud-auth/auth/login → /api/cloud-auth/auth/login
  然后走 Vite 代理 → {VITE_SERVE}/cloud-auth/auth/login
```

## 二、问题诊断

### 2.1 缺失的环境变量文件

**问题：** 项目中没有 `.env.development` 或 `.env.production` 文件

**根据 README.md 期望的环境变量：**

```env
# 后端 API 地址（网关地址）
VITE_SERVE=http://localhost:9527

# 前端地址
VITE_FRONTEND_URL=http://localhost:99

# 音乐页面地址
VITE_MUSIC_FRONTEND_URL=http://localhost:3000
```

**实际使用的环境变量：**

| 变量名 | 使用位置 | 用途 |
|--------|---------|------|
| `VITE_SERVE` | vite.config.ts:91 | 代理目标地址 |
| `VITE_SERVE` | Login/index.vue:108,113 | 第三方登录回调地址 |
| `VITE_SERVE` | Setting/index.vue:57 | 头像上传地址（生产环境） |
| `VITE_APP_BASE_API` | http.ts:13 | axios baseURL |
| `VITE_MUSIC_BASE_API` | http.ts:28 | 音乐 API 特殊处理 |
| `VITE_YIYAN_API` | http.ts:33 | 一言 API 特殊处理 |

### 2.2 Vite 开发服务器端口不一致

| 位置 | 端口 |
|------|------|
| `vite.config.ts:87` | `port: 5199` |
| `README.md:94` | `http://localhost:99` |

### 2.3 网关路由配置分析

**当前网关配置：**

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `server.port` | 9527 | 网关端口 |
| `discovery.locator.enabled` | true | 开启动态路由 |
| `lower-case-service-id` | true | 服务名自动转小写 |

**动态路由规则（基于 Consul 服务发现）：**

```
请求路径: /{服务名}/{路径}
网关转发: {服务名} 服务的 /{路径}

示例：
  /cloud-auth/auth/login → cloud-auth 服务 /auth/login
  /cloud-auth/auth/register → cloud-auth 服务 /auth/register
  /user/auth/info → user 服务 /auth/info（如果存在 user-service）
  /article/list → article 服务 /list（如果存在 article-service）
```

**当前问题：**
- cloud-web 服务运行在 8000 端口，但前端调用的是 `/article/list`、`/user/auth/info` 等路径
- 如果 Consul 中没有注册 `article`、`user` 等服务名，网关无法路由
- 需要确认 cloud-web 是否注册了这些路径，或者需要在网关配置静态路由

### 2.4 网关安全配置

**SecurityConfig.java 白名单：**

```java
.pathMatchers("/actuator/**", "/cloud-auth/**").permitAll()
.anyExchange().authenticated()  // 其他所有请求必须带 Token
```

**问题：**
- `/cloud-auth/**` 已放行，登录/注册可以访问
- 但 `/article/list`、`/user/auth/info` 等路径需要认证
- 未登录用户访问首页文章列表会被拒绝

### 阶段 2：修改 vite.config.ts 代理配置

**文件：** `frontend/blog/vite.config.ts`

**当前配置（第 86-100 行）：**

```typescript
server: {
    port: 5199,
    host: '0.0.0.0',
    proxy: {
        '/api': {
            target: `${loadEnv(mode, process.cwd()).VITE_SERVE}`,
            changeOrigin: true,
            rewrite: (path) => path.replace(/^\/api/, '')
        },
    }
}
```

**问题分析：**

当前代理逻辑：
- 前端请求 `/api/cloud-auth/auth/login`
- 代理重写路径（去掉 `/api`）→ `/cloud-auth/auth/login`
- 转发到 `VITE_SERVE`（网关）

这个逻辑是正确的，但需要确认：
1. 前端 API URL 是否都加了 `/api` 前缀（通过 `baseURL`）
2. 网关是否正确处理 `/cloud-auth/*` 路径

**建议修改点：**

```typescript
server: {
    port: 5199,  // 保持当前端口，与 README 不一致但功能正常
    host: '0.0.0.0',
    proxy: {
        '/api': {
            target: 'http://localhost:9527',  // 直接写死网关地址，避免环境变量问题
            changeOrigin: true,
            rewrite: (path) => path.replace(/^\/api/, '')
        },
    }
}
```

### 阶段 3：确认 axios baseURL 配置

```typescript
baseURL: import.meta.env.VITE_APP_BASE_API ?? '/',
```

**问题分析：**

### 阶段 4：梳理 API 调用路径与网关路由映射

**需要确认的路由映射：**

| 前端 URL | 经代理后 | 网关目标服务 | 服务端口 | 状态 |
|---------|---------|-------------|---------|------|
| `/api/cloud-auth/auth/login` | `/cloud-auth/auth/login` | cloud-auth | 9123 | ✅ 已放行 |
| `/api/cloud-auth/auth/register` | `/cloud-auth/auth/register` | cloud-auth | 9123 | ✅ 已放行 |
| `/api/cloud-auth/auth/send-code` | `/cloud-auth/auth/send-code` | cloud-auth | 9123 | ✅ 已放行 |
| `/api/user/logout` | `/user/logout` | ? | ? | ❓ 待确认 |
| `/api/user/auth/info` | `/user/auth/info` | ? | ? | ❓ 待确认 |
| `/api/article/list` | `/article/list` | ? | ? | ❓ 待确认 |
| `/api/comment/getComment` | `/comment/getComment` | ? | ? | ❓ 待确认 |

**问题：** 网关只开启动态路由，但没有配置静态路由

**解决方案：** 需要在网关配置中添加静态路由，将 `/user/*`、`/article/*` 等路径路由到 `cloud-web` 服务

### 阶段 5：修改网关路由配置

**文件：** `cloud-gatway9527/src/main/resources/application.yaml`

**需要添加的静态路由配置：**

```yaml
spring:
  cloud:
    gateway:
      routes:
        # cloud-web 服务路由
        - id: cloud-web-route
          uri: lb://cloud-web  # 使用 Consul 服务发现
          predicates:
            - Path=/article/**,/comment/**,/user/**,/favorite/**,/like/**,/link/**,/leaveWord/**,/photo/**,/video/**,/websiteInfo/**,/banners/**,/oauth/**

        # cloud-auth 服务路由（保留，确保认证相关路径）
        - id: cloud-auth-route
          uri: lb://cloud-auth
          predicates:
            - Path=/cloud-auth/**
```

**或者使用 URL 直连（如果 Consul 不可用）：**

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: cloud-web-route
          uri: http://localhost:8000
          predicates:
            - Path=/article/**,/comment/**,/user/**,/favorite/**,/like/**,/link/**,/leaveWord/**,/photo/**,/video/**,/websiteInfo/**,/banners/**,/oauth/**

        - id: cloud-auth-route
          uri: http://localhost:9123
          predicates:
            - Path=/cloud-auth/**
```

### 阶段 6：修改网关白名单配置

**文件：** `cloud-gatway9527/src/main/java/com/overthinker/cloud/gateway/config/SecurityConfig.java`

**当前白名单：**

```java
    "/video/**",        // 视频（公开）

```

### 阶段 7：统一硬编码 URL 处理

**文件：** `frontend/blog/src/views/Welcome/Login/index.vue`

**当前代码（第 108、113 行）：**

```vue
<el-link :href="env.MODE === 'development' ? env.VITE_SERVE + '/oauth/gitee/render' : env.VITE_SERVE + '/api/oauth/gitee/render'">
<el-link :href="env.MODE === 'development' ? env.VITE_SERVE + '/oauth/github/render' : env.VITE_SERVE + '/api/oauth/github/render'">
```

**问题：** 第三方登录 OAuth 回调路径不一致

**分析：**
- 开发环境：`VITE_SERVE/oauth/gitee/render` → 直接访问网关
- 生产环境：`VITE_SERVE/api/oauth/gitee/render` → 多了 `/api` 前缀

**解决方案：**

```vue
<!-- 统一使用 VITE_SERVE + 实际路径（网关路径，不需要 /api 前缀） -->
<el-link :href="env.VITE_SERVE + '/oauth/gitee/render'">
<el-link :href="env.VITE_SERVE + '/oauth/github/render'">
```

**文件：** `frontend/blog/src/views/Setting/index.vue`

**当前代码（第 57 行）：**

```typescript
const uploadAvatar = env.MODE === 'development' ? '/api/user/auth/upload/avatar' : env.VITE_SERVE + '/api/user/auth/upload/avatar'
```

**问题：**
- 开发环境：`/api/user/auth/upload/avatar` → 走 Vite 代理
- 生产环境：`VITE_SERVE/api/user/auth/upload/avatar` → 多了 `/api` 前缀

**解决方案：**

```typescript
// 开发环境：通过代理
// 生产环境：直接访问网关（不带 /api 前缀）
const uploadAvatar = env.MODE === 'development' 
  ? '/api/user/auth/upload/avatar' 
  : env.VITE_SERVE + '/user/auth/upload/avatar'
```
