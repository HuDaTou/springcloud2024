# 媒体资源分离迁移计划

## 一、项目概述

### 1.1 目标
将 `cloud-web` 服务中的文件上传操作迁移到使用 Feign 调用 `cloud-Media-Asset-Processing` 服务，实现媒体资源的统一管理和微服务架构的职责分离。

### 1.2 涉及服务
- **源服务**: `cloud-web` - 当前直接处理文件上传
- **目标服务**: `cloud-Media-Asset-Processing` - 专门处理媒体资源的服务
- **Feign 客户端**: `cloud-api` - 提供 Feign 接口定义

### 1.3 迁移范围
- 文章相关：封面、图片上传
- Banner 管理：图片上传
- 照片管理：照片上传
- 视频管理：视频、封面上传

---

## 二、当前架构分析

### 2.1 当前实现方式
`cloud-web` 服务直接使用 MinIO SDK 进行文件上传，存在以下问题：
- ❌ 职责不清晰：业务服务直接处理文件存储
- ❌ 代码重复：多个 Service 使用不同的 UploadUtils
- ❌ 维护困难：文件存储逻辑分散在多个服务中
- ❌ 扩展性差：难以统一管理媒体资源

## 五、详细迁移步骤

### 5.1 ArticleServiceImpl 迁移

#### 步骤 1：注入 Feign 客户端
```java
// ArticleServiceImpl.java
private final MediaClient mediaClient;
```

#### 步骤 2：迁移 uploadArticleCover()
**原实现**：
```java
@Override
public ResultData<String> uploadArticleCover(MultipartFile articleCover) {
    try {
        String articleCoverUrl = fileUploadUtils.uploadImage(UploadEnum.ARTICLE_COVER, articleCover);
        return ResultData.success(articleCoverUrl);
    } catch (FileUploadException e) {
        return ResultData.failure(e.getMessage());
    }
}
```

**新实现**：
```java
@Override
public ResultData<String> uploadArticleCover(MultipartFile articleCover) {
    try {
        // 1. 构建上传请求
        PresignedUploadRequest request = new PresignedUploadRequest(
            articleCover.getOriginalFilename(),
            articleCover.getSize(),
            articleCover.getContentType(),
            null, // MD5 可选
            "ARTICLE_COVER" // 文件类型
        );

        // 2. 获取预签名URL
        Long userId = SecurityUtils.getUserId();
        ResultData<Map<String, Object>> presignResult = mediaClient.getPresignedUploadUrl(userId, request);

        if (!presignResult.isSuccess()) {
            return ResultData.failure("获取上传URL失败");
        }

        Map<String, Object> data = presignResult.getData();
        String presignedUrl = (String) data.get("presignedUrl");
        String objectName = (String) data.get("objectName");

        // 3. 使用预签名URL上传文件到MinIO
        uploadToMinioWithPresignedUrl(presignedUrl, articleCover);

        // 4. 完成上传
        ResultData<Long> completeResult = mediaClient.completeUpload(objectName);
        if (!completeResult.isSuccess()) {
            return ResultData.failure("完成上传失败");
        }

        // 5. 获取文件访问URL
        ResultData<String> urlResult = mediaClient.getFileUrl(objectName);
        return urlResult;
    } catch (Exception e) {
        log.error("文章封面上传失败", e);
        return ResultData.failure("文章封面上传失败");
    }
}
```

#### 步骤 3：迁移 uploadArticleImage()
类似 uploadArticleCover() 的迁移方式，使用 `mediaClient.getPresignedUploadUrl()`。

#### 步骤 4：迁移 deleteArticleCover()
**原实现**：
```java
@Override
public ResultData<Void> deleteArticleCover(String articleCoverUrl) {
    String articleCoverName = articleCoverUrl.substring(articleCoverUrl.indexOf(bucketName) + bucketName.length());
    fileUploadUtils.deleteFiles(List.of(articleCoverName));
    return ResultData.success();
}
```

**新实现**：
```java
@Override
public ResultData<Void> deleteArticleCover(String articleCoverUrl) {
    try {
        // 从URL中提取objectName
        String objectName = extractObjectNameFromUrl(articleCoverUrl);

        // 调用 Feign 删除文件
        Long userId = SecurityUtils.getUserId();
        ResultData<Void> deleteResult = mediaClient.deleteFile(userId, objectName);

        return deleteResult;
    } catch (Exception e) {
        log.error("删除文章封面失败", e);
        return ResultData.failure();
    }
}
```

```java
@Override
public ResultData<Void> uploadPhoto(MultipartFile file, String name, Long parentId) {
    try {
        // 1. 检查重复
        if (photoMapper.selectCount(...) > 0) {
            return ResultData.failure("照片名称存在重复");
        }

        // 2. 构建上传路径（相册路径）
        String albumPath = buildAlbumPath(photoMapper, parentId);

        // 3. 构建上传请求
        PresignedUploadRequest request = new PresignedUploadRequest(
            name,
            file.getSize(),
            file.getContentType(),
            null,
            "PHOTO_ALBUM"
        );

        // 4. 获取预签名URL
        Long userId = SecurityUtils.getUserId();
        ResultData<Map<String, Object>> presignResult = mediaClient.getPresignedUploadUrl(userId, request);

        // 5. 上传文件
        uploadToMinioWithPresignedUrl(presignResult.getData().get("presignedUrl"), file);

        // 6. 完成上传
        mediaClient.completeUpload(presignResult.getData().get("objectName"));

        // 7. 保存数据库记录
        String photoUrl = mediaClient.getFileUrl(presignResult.getData().get("objectName")).getData();
        photoMapper.insert(new Photo()
            .setUserId(userId)
            .setParentId(parentId)
            .setName(name)
            .setUrl(photoUrl)
            .setType(AlbumOrPhotoEnum.PHOTO.getCode())
            .setSize(convertFileSizeToMB(file.getSize())));

        return ResultData.success();
    } catch (Exception e) {
        log.error("照片上传失败", e);
        return ResultData.failure(e.getMessage());
    }
}
```

```java
@Override
public Map<String, Object> uploadVideo(MultipartFile videoFile) {
    try {
        // 1. 参数校验
        validateVideoFile(videoFile);

        // 2. 初始化分片上传
        InitiateMultipartUploadRequest request = new InitiateMultipartUploadRequest(
            videoFile.getOriginalFilename(),
            videoFile.getSize(),
            videoFile.getContentType(),
            null,
            "VIDEO"
        );

        Long userId = SecurityUtils.getUserId();
        ResultData<Map<String, Object>> initiateResult = mediaClient.initiateMultipartUpload(userId, request);

        if (!initiateResult.isSuccess()) {
            throw new RuntimeException("初始化分片上传失败");
        }

        Map<String, Object> data = initiateResult.getData();
        String uploadId = (String) data.get("uploadId");
        Map<String, String> partUrls = (Map<String, String>) data.get("partUrls");

        // 3. 分片上传
        uploadPartsToMinio(partUrls, videoFile);

        // 4. 完成分片上传
        ResultData<Long> completeResult = mediaClient.completeMultipartUpload(uploadId);

        // 5. 获取视频URL
        String objectName = (String) data.get("objectName");
        ResultData<String> urlResult = mediaClient.getFileUrl(objectName);

        return Map.of(
            "video", urlResult.getData(),
            "videoSize", convertVideoSize(videoFile.getSize()),
            "videoType", videoFile.getContentType()
        );
    } catch (Exception e) {
        log.error("视频上传失败", e);
        throw new RuntimeException("视频上传失败", e);
    }
}
```

创建统一的工具类处理预签名URL上传：

```java
// MinioPresignedUploadHelper.java
@Component
public class MinioPresignedUploadHelper {

    /**
     * 使用预签名URL上传文件到MinIO
     */
    public void uploadWithPresignedUrl(String presignedUrl, MultipartFile file) {
        try {
            OkHttpClient client = new OkHttpClient();
            RequestBody requestBody = RequestBody.create(
                file.getBytes(),
                MediaType.parse(file.getContentType())
            );

            Request request = new Request.Builder()
                .url(presignedUrl)
                .put(requestBody)
                .build();

            Response response = client.newCall(request).execute();
            if (!response.isSuccessful()) {
                throw new RuntimeException("上传失败: " + response.code());
            }
        } catch (Exception e) {
            throw new RuntimeException("使用预签名URL上传失败", e);
        }
    }

     * 分片上传
     */

    }
}
```

### 6.2 URL 解析工具
```java
// MediaUrlUtils.java
@Component
public class MediaUrlUtils {

    /**
     * 从完整URL中提取objectName
     */
    public String extractObjectName(String fullUrl, String bucketName) {
        int bucketIndex = fullUrl.indexOf(bucketName);
        if (bucketIndex == -1) {
            throw new IllegalArgumentException("URL中未找到bucketName");
        }
        return fullUrl.substring(bucketIndex + bucketName.length() + 1);
    }
}
```
