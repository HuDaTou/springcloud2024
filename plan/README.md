## 🎯 项目目标

将 `cloud-web` 服务中的文件上传操作迁移到使用 Feign 调用 `cloud-Media-Asset-Processing` 服务，实现：

- ✅ **职责分离**：文件操作统一由媒体服务处理
- ✅ **架构优化**：cloud-web 不直接依赖 MinIO SDK
- ✅ **易于维护**：统一的文件管理逻辑
- ✅ **性能提升**：支持大文件分片上传和并发处理

---

## 📊 项目范围

### 涉及服务

| 服务 | 当前职责 | 目标职责 |
|------|---------|---------|
| cloud-web | 直接处理文件上传 | 通过 Feign 调用媒体服务 |
| cloud-Media-Asset-Processing | - | 统一处理所有文件操作 |
| cloud-api | - | 提供 Feign 客户端接口 |

### 迁移内容

| Service | 方法 | 文件类型 | 复杂度 |
|---------|------|---------|--------|
| ArticleServiceImpl | uploadArticleCover<br>uploadArticleImage<br>deleteArticleCover | 文章封面、图片 | ⭐ 简单 |
| BannersServiceImpl | uploadBannerImage<br>removeBannerById | Banner图片 | ⭐⭐ 中等 |
| PhotoServiceImpl | uploadPhoto<br>deletePhotoOrAlbum | 照片 | ⭐⭐⭐ 复杂 |
| VideoServiceImpl | uploadVideo<br>uploadVideoCover | 视频、封面 | ⭐⭐⭐⭐ 最复杂 |

### 常见问题

**Q: 为什么要迁移？**
A: 实现微服务职责分离，提高系统可维护性和扩展性。

**Q: 性能会受影响吗？**
A: 不会。通过预签名URL直接上传到MinIO，无中间转发，性能反而可能提升。

**Q: 大文件如何处理？**
A: 使用分片上传，支持断点续传和并行处理。
