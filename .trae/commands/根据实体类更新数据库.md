# 根据实体类更新数据库

## 技能目标

根据项目中的 Java 实体类（PO）自动分析表结构，生成对应的 SQL DDL 语句，并连接 PostgreSQL 数据库执行表结构更新操作。支持新增表、修改字段、删除字段等操作。

## 输入要求

- **实体类文件路径**：需要更新的实体类 Java 文件的绝对路径，例如：`/root/code/java/gitcode/springcloud2024/cloud-auth/src/main/java/com/overthinker/cloud/auth/entity/PO/User.java`
- **数据库 schema（可选）**：目标数据库 schema 名称，默认为 `auth`

## 执行步骤

1. **解析实体类**：读取并解析指定的 Java 实体类文件，提取 `@TableName`、`@TableId`、`@TableField` 等 MyBatis-Plus 注解信息
2. **生成 SQL DDL**：根据实体类字段类型和注解，生成 CREATE TABLE 或 ALTER TABLE 语句
3. **连接数据库**：使用项目配置的数据库连接信息（从 `.env` 文件读取）连接 PostgreSQL
4. **执行更新**：检查数据库中是否已存在该表，存在则执行 ALTER，不存在则执行 CREATE
5. **返回结果**：返回执行结果和影响的表信息

## 输出格式

```json
{
  "status": "success",
  "tableName": "sys_user",
  "operation": "CREATE/ALTER",
  "sql": "生成的 SQL 语句",
  "message": "执行结果描述"
}
```

## 错误处理

- 如果用户未提供实体类文件路径，提示："请提供实体类文件的绝对路径，例如：/root/code/java/gitcode/springcloud2024/cloud-auth/src/main/java/com/overthinker/cloud/auth/entity/PO/User.java"
- 如果实体类文件不存在，提示："指定的实体类文件不存在，请检查路径是否正确"
- 如果实体类未使用 `@TableName` 注解，提示："实体类必须使用 @TableName 注解指定表名"
- 如果数据库连接失败，提示："数据库连接失败，请检查 PostgreSQL 服务是否运行"
- 如果用户的需求超出技能范围（如非 MyBatis-Plus 实体类），提示："抱歉，此技能仅支持使用 MyBatis-Plus 注解的实体类，我可以帮您分析实体类结构"

## 支持的字段类型映射

| Java 类型 | PostgreSQL 类型 |
|-----------|----------------|
| Long | BIGINT |
| Integer | INTEGER |
| String | VARCHAR(255) |
| Boolean | BOOLEAN |
| LocalDateTime | TIMESTAMP |
| Date | DATE |
| Double | DOUBLE PRECISION |
| BigDecimal | DECIMAL |
| Byte[] | BYTEA |

## 注意事项

1. 执行前请确保 PostgreSQL 服务已启动（可通过 `docker ps | grep postgres` 检查）
2. 此操作会修改数据库结构，请在测试环境验证后再用于生产环境
3. 默认使用项目根目录 `.env` 文件中的数据库配置（POSTGRES_USER、POSTGRES_PASSWORD）
4. 如需回滚，请手动执行相应的 DROP TABLE 或 ALTER TABLE 语句
