# Spring Cloud Java 后端项目开发规范

> **适用范围**：本规范适用于所有 Java 后端项目模块。
>
> - ✅ 适用：`cloud-auth/`、`cloud-system/`、`cloud-gatway9527/`、`cloud-common/` 等所有 `cloud-*/` 目录
> - ❌ 不适用：`frontend/` 前端项目目录

---

## 1. 命名规范

### 文件命名

❌ **禁止**：
- 拼写错误：`PhotoUploudUtils.java` → `PhotoUploadUtils.java`
- 中文文件名
- 特殊字符

✅ **正确**：
```
UserController.java
UserService.java
UserServiceImpl.java
UserMapper.java
User.java / UserPO.java
UserDTO.java / UserVO.java
```

### 类命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 接口 | 业务名称 | `UserService`, `AuthService` |
| 实现类 | 接口名 + `Impl` | `UserServiceImpl` |
| 控制器 | 业务名 + `Controller` | `UserController` |
| Mapper | 实体名 + `Mapper` | `UserMapper` |
| DTO/VO | 业务名 + 后缀 | `UserDTO`, `UserVO` |
| 常量类 | 业务名 + `Const` | `AuthRedisConst` |
| 工具类 | 业务名 + `Utils` | `DateUtils` |
| 枚举类 | 业务名 + `Enum` | `StatusEnum` |
| 异常类 | 业务名 + `Exception` | `BusinessException` |
| 配置类 | 业务名 + `Config` | `SecurityConfig` |

### 方法命名

| 场景 | 前缀 | 示例 |
|------|------|------|
| 获取单个 | `get` | `getUserById(Long id)` |
| 获取列表 | `list` | `listUsers()` |
| 分页查询 | `page` | `pageUsers(PageParams params)` |
| 新增 | `save`/`add`/`create` | `saveUser(User user)` |
| 更新 | `update` | `updateUser(Long id, User user)` |
| 删除 | `delete`/`remove` | `deleteUser(Long id)` |
| 检查 | `check`/`verify`/`is` | `checkUserExist(String username)` |
| 转换 | `convert`/`to` | `convertToVO(User user)` |
| 统计 | `count` | `countUsers()` |

### 变量命名

- ✅ 小驼峰：`userName`, `createTime`
- ✅ 布尔变量：`isDeleted`, `hasPermission`
- ❌ 避免拼音：`yongHuMing` → `userName`
- ❌ 避免歧义：`data` → `userData`

### 常量命名

- ✅ 全大写 + 下划线：`MAX_LOGIN_ATTEMPTS`, `TOKEN_EXPIRE_TIME`

## 2. 日期时间规范

### 核心原则

**必须使用 `java.time` API，禁止使用旧 API**

### 禁止使用

| 禁止类型 | 推荐替代 |
| -------- | -------- |
| `java.util.Date` | `LocalDateTime` |
| `java.util.Calendar` | `LocalDateTime` |
| `SimpleDateFormat` | `DateTimeFormatter` |
| `java.sql.Timestamp` | `LocalDateTime` |

### 推荐使用

| 类型 | 说明 |
| ---- | ---- |
| `LocalDateTime` | 本地日期时间（实体类字段） |
| `LocalDate` | 本地日期（仅日期） |
| `LocalTime` | 本地时间（仅时间） |
| `Instant` | 时间戳 |
| `DateTimeFormatter` | 格式化（线程安全） |

### 代码示例

```java
// 获取当前时间
LocalDateTime now = LocalDateTime.now();

// 格式化
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String formatted = now.format(formatter);

// 解析
LocalDateTime parsed = LocalDateTime.parse(str, formatter);

// 日期计算
LocalDateTime tomorrow = now.plusDays(1);
LocalDateTime lastYear = now.minusYears(1);
```

## 3. 工具类规范

### 统一位置

- 公共工具类：`cloud-common/cloud-common-core/.../utils/`
- 模块工具类：`{module}/.../utils/`

### 设计原则

```java
public class MyStringUtils {

    private MyStringUtils() { }

    // 所有方法必须是 public static
    public static boolean isEmpty(String str) {
        return str == null || str.trim().isEmpty();
    }
}
```

### 工具类列表

| 工具类 | 用途 | 说明 |
|--------|------|------|
| `MyDateUtils` | 日期时间 | 基于 java.time |
| `MyStringUtils` | 字符串 | 基于 Apache Commons Lang3 |
| `MyJsonUtils` | JSON | 基于 Fastjson2 |

## 4. 异常处理规范

### 异常层次

```
Exception
└── RuntimeException
    ├── BusinessException (业务异常)
    ├── ValidationException (校验异常)
    └── FileUploadException (文件上传异常)
```

### 自定义异常

```java
@Getter
public class BusinessException extends RuntimeException {
    private final ReturnCodeEnum returnCodeEnum;
    private final Object data;

    public BusinessException(ReturnCodeEnum returnCodeEnum) {
        super(returnCodeEnum.getMessage());
        this.returnCodeEnum = returnCodeEnum;
        this.data = null;
    }
}
```

### 全局异常处理

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ResultData<Object> handleBusinessException(BusinessException e) {
        log.warn("业务异常：{}", e.getMessage());
        return ResultData.failure(e.getReturnCodeEnum(), e.getData());
    }
}
```

### 异常使用

| 场景 | 异常类型 | 示例 |
|------|----------|------|
| 参数校验失败 | `ValidationException` | `throw new ValidationException("用户名不能为空")` |
| 业务规则不满足 | `BusinessException` | `throw new BusinessException(ReturnCodeEnum.RC400, "用户已存在")` |

## 5. 常量规范

### 常量类组织

```
constants/
├── HttpConst.java          # HTTP 相关
├── NumberConst.java        # 数字常量
├── AuthRedisConst.java     # Redis Key 常量
└── ...
```

### 常量定义

```java
public class NumberConst {
    public static final int ZERO = 0;
    public static final int DEFAULT_PAGE_SIZE = 10;
    public static final int MAX_PAGE_SIZE = 100;

```

### 避免魔法值

❌ **禁止**：
```java
if (user.getStatus() == 1) { ... }
Thread.sleep(30000);
```

✅ **推荐**：使用枚举或常量类

## 6. 日志规范

### 日志工具

✅ **必须使用**：SLF4J + Lombok `@Slf4j`

### 日志级别使用

| 级别 | 使用场景 |
|------|----------|
| `trace` | 详细跟踪信息 |
| `debug` | 调试信息，生产环境关闭 |
| `info` | 一般信息，重要业务流程 |
| `warn` | 警告信息，业务异常 |
| `error` | 错误信息，系统异常 |

### 正确示例

```java
log.info("用户登录成功，用户名：{}", username);
log.warn("用户登录失败，原因：{}", reason);
log.error("数据库连接失败", e);
```

### 禁止行为

❌ **禁止**：在日志中输出敏感信息
❌ **禁止**：`System.out.println()` 打印日志

## 8. 代码组织规范

### 包结构

```
com.overthinker.cloud.{module}/
├── Application.java              # 启动类
├── config/                       # 配置类
├── controller/                   # 控制器
├── service/                      # 服务层
│   └── impl/                     # 实现类
├── mapper/                       # 数据访问层
├── entity/                       # 实体类
│   ├── PO/                       # 持久对象
│   ├── DTO/                      # 数据传输对象
│   └── VO/                       # 视图对象
├── listener/                     # 事件监听器
├── handler/                      # 处理器
├── constants/                    # 常量
├── utils/                        # 工具类
└── exception/                    # 异常类
```

### 导入顺序

```java
// 1. Java 标准库
import java.util.List;

// 2. 第三方库
import org.springframework.stereotype.Service;
import lombok.RequiredArgsConstructor;

// 3. 项目内部包
import com.overthinker.cloud.common.core.resp.ResultData;
```

### 类成员顺序

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class UserServiceImpl implements UserService {

    private static final int MAX_RETRY_TIMES = 3;

    private static final ThreadLocal<User> CURRENT_USER = new ThreadLocal<>();

    // 3. 实例变量（依赖注入）
    private final UserMapper userMapper;

    // 4. 公共方法
    @Override
    public User getById(Long id) {
        // ...
    }

    // 5. 私有方法
    private void validateUser(User user) {
        // ...
    }
}
```
