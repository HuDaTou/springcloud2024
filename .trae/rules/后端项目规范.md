## 1. 命名规范

### 1.1 文件命名

#### 1.1.1 禁止
- ❌ 拼写错误：`PhotoUploudUtils.java` → `PhotoUploadUtils.java`
- ❌ 中文文件名
- ❌ 特殊字符（除了 `_` 和 `-`，推荐使用驼峰）

#### 1.1.2 正确示例
```
UserController.java
UserService.java
UserServiceImpl.java
UserMapper.java
User.java
UserDTO.java
UserVO.java
UserPO.java
```

### 1.2 类命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 接口 | 业务名称（不需要 `I` 前缀） | `UserService`, `AuthService` |
| 实现类 | 接口名 + `Impl` | `UserServiceImpl`, `AuthServiceImpl` |
| 控制器 | 业务名 + `Controller` | `UserController`, `AuthController` |
| Mapper | 实体名 + `Mapper` | `UserMapper`, `RoleMapper` |
| 实体类 | 业务名（或加 `PO` 后缀） | `User` 或 `UserPO` |
| DTO | 业务名 + `DTO` | `UserDTO`, `UserCreateDTO` |
| VO | 业务名 + `VO` | `UserVO`, `UserDetailVO` |
| 常量类 | 业务名 + `Const` | `AuthRedisConst`, `SecurityConst` |
| 工具类 | 业务名 + `Utils` | `DateUtils`, `StringUtils` |
| 枚举类 | 业务名 + `Enum` | `ReturnCodeEnum`, `StatusEnum` |
| 异常类 | 业务名 + `Exception` | `BusinessException`, `FileUploadException` |
| 配置类 | 业务名 + `Config` | `SecurityConfig`, `MinioConfig` |
| 监听器类 | 业务名 + `Listener` | `EmailListener`, `PermissionListener` |
| 处理器类 | 业务名 + `Handler` | `GlobalExceptionHandler`, `MyMetaObjectHandler` |

### 1.3 方法命名

| 场景 | 前缀 | 示例 |
|------|------|------|
| 获取单个对象 | `get` | `getUserById(Long id)` |
| 获取列表 | `list` | `listUsers()` |
| 分页查询 | `page` | `pageUsers(PageParams params)` |
| 新增 | `save`/`add`/`create` | `saveUser(User user)` |
| 更新 | `update` | `updateUser(User user)` |
| 删除 | `delete`/`remove` | `deleteUser(Long id)` |
| 检查 | `check`/`verify`/`is` | `checkUserExist(String username)`, `isEmailValid(String email)` |
| 转换 | `convert`/`to` | `convertToVO(User user)`, `toDTO(User user)` |
| 初始化 | `init` | `initConfig()` |
| 统计 | `count` | `countUsers()` |
| 构建 | `build` | `buildUserVO(User user)` |

### 1.4 变量命名

- ✅ 使用小驼峰：`userName`, `createTime`
- ✅ 避免单字母变量（循环除外：`i`, `j`, `k`）
- ✅ 布尔变量使用 `is`/`has`/`can` 前缀：`isDeleted`, `hasPermission`, `canEdit`
- ❌ 避免使用拼音：`yongHuMing` → `userName`
- ❌ 避免歧义：`data` → `userData` 或 `orderList`

### 1.5 常量命名

- ✅ 全大写 + 下划线分隔
```java
public static final String DEFAULT_USERNAME = "admin";
public static final int MAX_LOGIN_ATTEMPTS = 5;
public static final long TOKEN_EXPIRE_TIME = 7200;
```

## 2. 日期时间规范

### 2.1 核心原则

**本项目所有模块必须使用 `java.time` 包下的日期时间 API，禁止使用旧的日期时间 API。

### 2.2 禁止使用的类型

| 禁止类型 | 说明 | 推荐替代 |
| -------- | ---- | -------- |
| `java.util.Date` | 过时，线程不安全 | `LocalDateTime` / `LocalDate` |
| `java.util.Calendar` | 过时，线程不安全 | `LocalDateTime` / `LocalDate` |
| `java.util.GregorianCalendar` | 过时 | `LocalDateTime` |
| `java.util.TimeZone` | 过时 | `ZoneId` / `ZonedDateTime` |
| `java.sql.Date` | 过时 | `LocalDate` |
| `java.sql.Time` | 过时 | `LocalTime` |
| `java.sql.Timestamp` | 过时 | `LocalDateTime` |
| `java.text.SimpleDateFormat` | 线程不安全 | `DateTimeFormatter` |
| `java.text.DateFormat` | 线程不安全 | `DateTimeFormatter` |

### 2.3 推荐使用的类型

| 类型 | 说明 | 使用场景 |
| ---- | ---- | -------- |
| `LocalDateTime` | 本地日期时间，不带时区信息 | 业务实体类字段 |
| `LocalDate` | 本地日期，不带时间 | 业务实体类字段（仅日期） |
| `LocalTime` | 本地时间，不带日期 | 业务实体类字段（仅时间） |
| `ZoneId` | 时区 ID | 时区处理 |
| `DateTimeFormatter` | 日期时间格式化 | 线程安全的格式化 |
| `Duration` | 时间段（秒/纳秒） | 时间差计算 |
| `Period` | 时间段（年/月/日） | 日期间隔计算 |
| `Instant` | 时间戳 | 精确的时间点表示 |
| `ZonedDateTime` | 带时区的日期时间 | 多时区场景 |

### 2.4 代码示例

#### 2.4.1 获取当前时间

```java
// 不推荐
Date now = new Date();
Calendar calendar = Calendar.getInstance();

// 推荐
LocalDateTime now = LocalDateTime.now();
LocalDate today = LocalDate.now();
LocalTime currentTime = LocalTime.now();
```

#### 2.4.2 日期格式化

```java
// 不推荐
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
String formatted = sdf.format(date);

// 推荐
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String formatted = LocalDateTime.now().format(formatter);
```

#### 2.4.3 日期解析

```java
// 不推荐
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
Date date = sdf.parse(dateStr);

// 推荐
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
LocalDateTime dateTime = LocalDateTime.parse(dateStr, formatter);
```

#### 2.4.4 日期计算

```java
// 不推荐
Calendar calendar = Calendar.getInstance();
calendar.add(Calendar.DAY_OF_MONTH, 1);
Date tomorrow = calendar.getTime();

// 推荐
LocalDateTime tomorrow = LocalDateTime.now().plusDays(1);
LocalDateTime yesterday = LocalDateTime.now().minusDays(1);
LocalDateTime nextMonth = LocalDateTime.now().plusMonths(1);
LocalDateTime lastYear = LocalDateTime.now().minusYears(1);
```

#### 2.4.5 日期比较

```java
// 不推荐
if (date1.before(date2)) { ... }
if (date1.after(date2)) { ... }

// 推荐
if (dateTime1.isBefore(dateTime2));
if (dateTime1.isAfter(dateTime2));
if (dateTime1.isEqual(dateTime2));
```

#### 2.4.6 时间差计算

```java
// 不推荐
long diff = date2.getTime() - date1.getTime();
long days = diff / (1000 * 60 * 60 * 24);

// 推荐
long days = ChronoUnit.DAYS.between(dateTime1, dateTime2);
long hours = ChronoUnit.HOURS.between(dateTime1, dateTime2);
long minutes = ChronoUnit.MINUTES.between(dateTime1, dateTime2);
```

#### 2.4.7 获取指定时间

```java
// 推荐
LocalDateTime startTime = LocalDateTime.of(2024, 1, 15, 10, 30, 0);
LocalDate date = LocalDate.of(2024, 1, 15);
LocalTime time = LocalTime.of(10, 30);
```

### 2.5 实体类规范

所有实体类的日期时间字段必须使用 `LocalDateTime` 或 `LocalDate`。

```java
// 正确示例
@Data
@TableName("sys_user")
public class User extends BaseData {
    private Long id;
    private String username;
    // 正确使用 LocalDateTime
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    private LocalDateTime loginTime;
    // 如果只需要日期，使用 LocalDate
    private LocalDate birthday;
}

// 错误示例
@Data
@TableName("sys_user")
public class User {
    private Long id;
    private String username;
    // 错误！不要使用 Date
    private Date createTime;
    // 错误！不要使用 Timestamp
    private Timestamp updateTime;
}
```

项目中有两个日期时间工具类：

1. **推荐使用 `MyDateUtils` - 基于 `java.time` API


2. **禁止使用 `DateUtils` - 基于旧 API

确保 Jackson 能够正确序列化和反序列化 `java.time` 类型。

```java

    public ObjectMapper objectMapper() {

```

### 2.9 特殊情况处理

#### 2.9.1 与旧代码的互操作

如果必须与使用 `java.util.Date`，使用以下转换方式：

```java
// LocalDateTime -> Date
LocalDateTime localDateTime = LocalDateTime.now();
Date date = Date.from(localDateTime.atZone(ZoneId.systemDefault()).toInstant());

// Date -> LocalDateTime
Date date = new Date();
LocalDateTime localDateTime = LocalDateTime.ofInstant(date.toInstant(), ZoneId.systemDefault());

// LocalDate -> Date
LocalDate localDate = LocalDate.now();
Date date = Date.from(localDate.atStartOfDay(ZoneId.systemDefault()).toInstant());
```

#### 2.9.2 时间戳转换

```java
// LocalDateTime -> 时间戳（毫秒）
long timestamp = LocalDateTime.now().atZone(ZoneId.systemDefault()).toInstant().toEpochMilli();

// 时间戳（毫秒）-> LocalDateTime
LocalDateTime dateTime = LocalDateTime.ofInstant(
    Instant.ofEpochMilli(timestamp),
    ZoneId.systemDefault()
);
```

### 2.10 常用 DateTimeFormatter 模式

```java
// 预定义格式化器
DateTimeFormatter.ISO_LOCAL_DATE_TIME // 2024-01-15T10:30:00
DateTimeFormatter.ISO_LOCAL_DATE // 2024-01-15
DateTimeFormatter.ISO_LOCAL_TIME // 10:30:00
DateTimeFormatter.ISO_DATE_TIME // 2024-01-15T10:30:00+08:00

// 自定义格式化器
DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")
DateTimeFormatter.ofPattern("yyyy/MM/dd HH:mm:ss")
DateTimeFormatter.ofPattern("yyyyMMddHHmmss")
DateTimeFormatter.ofPattern("yyyy-MM-dd")
DateTimeFormatter.ofPattern("HH:mm:ss")
```

## 4. 异常处理规范

### 4.1 异常层次

```
Exception
├── RuntimeException
│   ├── BusinessException (业务异常)
│   ├── ValidationException (校验异常)
│   ├── FileUploadException (文件上传异常)
│   └── ...
└── ...
```

### 4.2 自定义异常设计

```java
@Getter
public class BusinessException extends RuntimeException {
    private final ReturnCodeEnum returnCodeEnum;
    private final Object data;

    public BusinessException(ReturnCodeEnum returnCodeEnum) {
        super(returnCodeEnum.getMessage());
        this.returnCodeEnum = returnCodeEnum;
        this.data = null;
    }

    public BusinessException(ReturnCodeEnum returnCodeEnum, String message) {
        super(message);
        this.returnCodeEnum = returnCodeEnum;
        this.data = null;
    }

    public BusinessException(ReturnCodeEnum returnCodeEnum, Object data) {
        super(returnCodeEnum.getMessage());
        this.returnCodeEnum = returnCodeEnum;
        this.data = data;
    }
}
```

### 4.3 异常使用场景

| 场景 | 异常类型 | 示例 |
|------|----------|------|
| 参数校验失败 | `ValidationException` | `throw new ValidationException("用户名不能为空")` |
| 业务规则不满足 | `BusinessException` | `throw new BusinessException(ReturnCodeEnum.RC400, "用户已存在")` |
| 外部接口调用失败 | `ExternalApiException` | `throw new ExternalApiException("支付接口调用失败")` |

### 4.4 全局异常处理

```java
@RestControllerAdvice
@Order(1)
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ResultData<Object> handleBusinessException(BusinessException e) {
        log.warn("业务异常：{}", e.getMessage());
        return ResultData.failure(e.getReturnCodeEnum(), e.getData());
    }

    @ExceptionHandler(ValidationException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ResultData<Object> handleValidationException(ValidationException e) {
        log.warn("参数校验失败：{}", e.getMessage());
        return ResultData.failure(ReturnCodeEnum.RC400, e.getMessage());
    }

    @ExceptionHandler(RuntimeException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ResultData<String> handleRuntimeException(RuntimeException e) {
        log.error("系统异常：{}", e.getMessage(), e);
        return ResultData.failure(ReturnCodeEnum.RC500, "系统内部错误");
    }
}
```

### 4.5 异常抛出规范

✅ **推荐**：
```java
public void checkUserExist(String username) {
    User user = userMapper.selectByUsername(username);
    if (user != null) {
        throw new BusinessException(ReturnCodeEnum.RC400, "用户名已存在");
    }
}
```

❌ **避免**：
```java
public void checkUserExist(String username) {
    User user = userMapper.selectByUsername(username);
    if (user != null) {
        throw new RuntimeException("用户名已存在"); // 太泛化
    }
}
```

## 6. 数据库操作规范

### 6.1 实体类设计

```java
@Data
@TableName("sys_user")
public class User extends BaseData {
    @TableId(value = "id", type = IdType.ASSIGN_ID)
    @Schema(description = "用户ID")
    private Long id;

    @TableField("username")
    @Schema(description = "用户名")
    private String username;

    // 继承自 BaseData
    // private LocalDateTime createTime;
    // private LocalDateTime updateTime;
    // private Boolean deleted;
}
```

### 6.2 Mapper 接口

```java
public interface UserMapper extends BaseMapper<User> {

    // ✅ 简单查询使用 LambdaQueryWrapper
    // ✅ 复杂查询可以自定义方法

    @Select("SELECT * FROM sys_user WHERE username = #{username}")
    User selectByUsername(@Param("username") String username);

    @Select("SELECT * FROM sys_user WHERE create_time >= #{startTime} AND create_time <= #{endTime}")
    List<User> selectByTimeRange(@Param("startTime") LocalDateTime startTime,
                                   @Param("endTime") LocalDateTime endTime);
}
```

### 6.3 Service 层

```java
@Service("userService")
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {

    @Override
    public User getById(Long id) {
        if (id == null) {
            throw new ValidationException("用户ID不能为空");
        }
        return this.getById(id);
    }
```

## 7. API 设计规范

### 7.1 URL 设计原则

- ✅ 使用小写字母和连字符：`/api/users`, `/api/user-profiles`
- ✅ 使用名词（复数）：`/api/users` 而不是 `/api/getUsers`
- ✅ 使用 HTTP 方法表示动作：
  - `GET` - 查询
  - `POST` - 创建
  - `PUT` - 完整更新
  - `PATCH` - 部分更新
  - `DELETE` - 删除

### 7.2 RESTful API 示例

| 功能 | 方法 | URL | 说明 |
|------|------|-----|------|
| 查询用户列表 | GET | `/api/users` | 查询所有用户 |
| 查询单个用户 | GET | `/api/users/{id}` | 根据ID查询 |
| 创建用户 | POST | `/api/users` | 创建新用户 |
| 更新用户 | PUT | `/api/users/{id}` | 完整更新用户信息 |
| 部分更新 | PATCH | `/api/users/{id}` | 仅更新部分字段 |
| 删除用户 | DELETE | `/api/users/{id}` | 删除用户 |

### 7.3 Controller 示例

```java
@RestController
@RequestMapping("/api/users")
@Tag(name = "用户管理", description = "用户相关接口")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    @GetMapping
    @Operation(summary = "查询用户列表")
    public ResultData<PageResult<UserVO>> listUsers(
        @Parameter(description = "分页参数") PageParams params

    @PostMapping
    @Operation(summary = "创建用户")
    public ResultData<Void> createUser(
        @Parameter(description = "用户信息") @RequestBody @Valid UserCreateDTO dto
    ) {
        userService.create(dto);
        return ResultData.success();
    }

    @PutMapping("/{id}")
    @Operation(summary = "更新用户")
    public ResultData<Void> updateUser(
        @Parameter(description = "用户ID") @PathVariable Long id,
        @Parameter(description = "用户信息") @RequestBody @Valid UserUpdateDTO dto
    ) {
        userService.update(id, dto);
        return ResultData.success();
    }

    @DeleteMapping("/{id}")
    @Operation(summary = "删除用户")
    public ResultData<Void> deleteUser(
        @Parameter(description = "用户ID") @PathVariable Long id
    ) {
        userService.delete(id);
        return ResultData.success();
    }
}
```

### 7.4 统一返回格式

```java
@Data
@Builder
@AllArgsConstructor
@NoArgsConstructor
public class ResultData<T> {

    @Schema(description = "状态码")
    private Integer code;

    @Schema(description = "消息")
    private String msg;

    @Schema(description = "数据")
    private T data;

    @Schema(description = "时间戳")
    private Long timestamp;

    public static <T> ResultData<T> success() {
        return success(null);
    }

    public static <T> ResultData<T> success(T data) {
        return ResultData.<T>builder()
            .code(ReturnCodeEnum.RC200.getCode())
            .msg(ReturnCodeEnum.RC200.getMessage())
            .data(data)
            .timestamp(System.currentTimeMillis())
            .build();
    }

    public static <T> ResultData<T> failure(ReturnCodeEnum returnCodeEnum) {
        return failure(returnCodeEnum, null);
    }

    public static <T> ResultData<T> failure(ReturnCodeEnum returnCodeEnum, T data) {
        return ResultData.<T>builder()
            .code(returnCodeEnum.getCode())
            .msg(returnCodeEnum.getMessage())
            .data(data)
            .timestamp(System.currentTimeMillis())
            .build();
    }
}
```

## 8. 日志规范

### 8.1 日志工具选择

✅ **推荐使用**：SLF4J + Lombok `@Slf4j`

```java
@Slf4j
@Service
public class UserServiceImpl implements UserService {

    @Override
    public User getById(Long id) {
        log.debug("查询用户，ID: {}", id);
        try {
            User user = userMapper.selectById(id);
            if (user == null) {
                log.warn("用户不存在，ID: {}", id);
            }
            return user;
        } catch (Exception e) {
            log.error("查询用户失败，ID: {}", id, e);
            throw e;
        }
    }
}
```

### 8.2 日志级别使用

| 级别 | 用途 | 示例 |
|------|------|------|
| `ERROR` | 错误，影响功能运行 | 接口调用失败、数据库操作失败 |
| `WARN` | 警告，需要关注但不影响运行 | 参数不规范、重复操作 |
| `INFO` | 关键业务流程 | 用户登录、订单创建 |
| `DEBUG` | 调试信息 | SQL 语句、请求参数 |
| `TRACE` | 更详细的调试信息 | 很少使用 |

### 8.3 日志格式

✅ **推荐**：使用占位符，而不是字符串拼接
```java
log.info("用户登录，用户名: {}, IP: {}", username, ip); // ✅
log.info("用户登录，用户名: " + username + ", IP: " + ip); // ❌
```

### 8.4 敏感信息处理

❌ **禁止**：在日志中输出敏感信息
```java
log.info("用户注册，用户名: {}, 密码: {}", username, password); // ❌ 不要输出密码
log.info("用户登录，token: {}", token); // ❌ 不要输出 token
```

### 8.5 异常日志

```java
try {
    // ...
} catch (Exception e) {
    // ✅ 记录完整异常堆栈
    log.error("操作失败，原因: {}", e.getMessage(), e);
    // ❌ 不要只记录消息
    // log.error("操作失败，原因: {}", e.getMessage());
}
```
