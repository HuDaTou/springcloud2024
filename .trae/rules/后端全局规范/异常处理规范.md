# 异常处理规范

## 一、异常目录结构

### 1. 统一存放位置

所有异常类必须存放在以下目录：

```
cloud-common/cloud-common-core/src/main/java/com/overthinker/cloud/common/core/exception/
```

### 2. 目录组织方式

```
exception/
├── BusinessException.java          # 业务异常（通用）
├── FileUploadException.java        # 文件上传异常
├── ServerException.java            # 服务器异常
├── MinioConnectionException.java   # Minio 连接异常
├── auth/                           # auth 模块私有异常
│   ├── AuthException.java
│   └── TokenExpiredException.java
├── system/                         # system 模块私有异常
│   └── ConfigException.java
└── {module}/                       # 其他模块私有异常目录
    └── {Module}Exception.java
```

### 3. 模块私有异常

- 各模块可在 `exception/` 目录下创建以模块名命名的子目录
- 子目录内存放该模块特有的异常类
- 例如：`auth/`、`system/`、`gateway/` 等

## 二、异常类设计规范

### 1. 强制要求（AI必须100%遵守）

1. 必须继承 `RuntimeException`，禁止继承 `Exception` 或 `Throwable`
2. 必须使用 `@Getter` 注解（Lombok），禁止手动编写 getter 方法
3. 必须包含 `ReturnCodeEnum` 字段，用于标识异常类型
4. 必须提供至少一个接收 `ReturnCodeEnum` 参数的构造函数
5. 必须在构造函数中调用 `super(returnCodeEnum.getMsg())` 设置异常消息
6. 字段必须使用 `final` 修饰，确保不可变
7. 类名必须以 `Exception` 结尾

### 2. 推荐做法

1. 推荐提供多个构造函数，支持不同场景：
   - 只传入 `ReturnCodeEnum`
   - 传入自定义消息 `String`
   - 传入 `ReturnCodeEnum` 和自定义消息
2. 推荐添加类注释，说明异常用途
3. 推荐使用 `@EqualsAndHashCode(callSuper = true)` 注解（如果需要）

### 3. 禁止行为

1. 禁止继承 `Exception`（必须继承 `RuntimeException`）
2. 禁止使用受检异常（checked exception）
3. 禁止在异常类中定义业务逻辑方法
4. 禁止省略 `ReturnCodeEnum` 字段
5. 禁止使用非 `final` 修饰字段

## 三、标准异常类模板

### 1. 基础异常类（BusinessException）

```java
package com.overthinker.cloud.common.core.exception;

import com.overthinker.cloud.common.core.resp.ReturnCodeEnum;
import lombok.Getter;

/**
 * 业务异常
 * <p>
 * 用于处理业务逻辑中的异常情况
 * </p>
 *
 * @author overthinker
 * @since 2024-01-15
 */
@Getter
public class BusinessException extends RuntimeException {

    /**
     * 返回码枚举
     */
    private final ReturnCodeEnum returnCodeEnum;

    /**
     * 构造函数 - 使用枚举默认消息
     *
     * @param returnCodeEnum 返回码枚举
     */
    public BusinessException(ReturnCodeEnum returnCodeEnum) {
        super(returnCodeEnum.getMsg());
        this.returnCodeEnum = returnCodeEnum;
    }

    /**
     * 构造函数 - 使用自定义消息
     *
     * @param returnCodeEnum 返回码枚举
     * @param msg            自定义异常消息
     */
    public BusinessException(ReturnCodeEnum returnCodeEnum, String msg) {
        super(msg);
        this.returnCodeEnum = returnCodeEnum;
    }
}
```

### 2. 特定场景异常类（FileUploadException）

```java
package com.overthinker.cloud.common.core.exception;

import com.overthinker.cloud.common.core.resp.ReturnCodeEnum;
import lombok.Getter;

/**
 * 文件上传异常
 * <p>
 * 用于处理文件上传过程中的异常情况
 * </p>
 *
 * @author overthinker
 * @since 2024-01-15
 */
@Getter
public class FileUploadException extends RuntimeException {

    /**
     * 返回码枚举
     */
    private final ReturnCodeEnum returnCodeEnum;

    /**
     * 构造函数 - 使用自定义消息
     *
     * @param message 自定义异常消息
     */
    public FileUploadException(String message) {
        super(message);
        this.returnCodeEnum = ReturnCodeEnum.FILE_UPLOAD_ERROR;
    }

    /**
     * 构造函数 - 使用自定义消息和返回码
     *
     * @param message       自定义异常消息
     * @param returnCodeEnum 返回码枚举
     */
    public FileUploadException(String message, ReturnCodeEnum returnCodeEnum) {
        super(message);
        this.returnCodeEnum = returnCodeEnum;
    }

    /**
     * 构造函数 - 使用枚举默认消息
     *
     * @param returnCodeEnum 返回码枚举
     */
    public FileUploadException(ReturnCodeEnum returnCodeEnum) {
        super(returnCodeEnum.getMsg());
        this.returnCodeEnum = returnCodeEnum;
    }
}
```

## 四、异常使用规范

### 1. 抛出异常

```java
// ✅ 推荐：使用枚举默认消息
throw new BusinessException(ReturnCodeEnum.USER_NOT_FOUND);

// ✅ 推荐：使用自定义消息
throw new BusinessException(ReturnCodeEnum.USER_NOT_FOUND, "用户ID: " + userId + " 不存在");

// ✅ 推荐：使用特定场景异常
throw new FileUploadException("文件大小超过限制：10MB");

// ❌ 禁止：直接抛出 RuntimeException
// throw new RuntimeException("用户不存在");
```

### 2. 捕获异常

```java
// ✅ 推荐：捕获并转换为业务异常
try {
    // 业务逻辑
} catch (IOException e) {
    throw new FileUploadException(ReturnCodeEnum.FILE_UPLOAD_ERROR, "文件读取失败: " + e.getMessage());
}

// ✅ 推荐：记录日志后抛出
try {
    // 业务逻辑
} catch (Exception e) {
    log.error("操作失败", e);
    throw new ServerException(ReturnCodeEnum.SERVER_ERROR);
}
```

### 3. 全局异常处理

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    /**
     * 处理业务异常
     */
    @ExceptionHandler(BusinessException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ResultData<Object> handleBusinessException(BusinessException e) {
        log.warn("业务异常：{}", e.getMessage());
        return ResultData.failure(e.getReturnCodeEnum());
    }

    /**
     * 处理文件上传异常
     */
    @ExceptionHandler(FileUploadException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ResultData<Object> handleFileUploadException(FileUploadException e) {
        log.warn("文件上传异常：{}", e.getMessage());
        return ResultData.failure(e.getReturnCodeEnum());
    }

    /**
     * 处理服务器异常
     */
    @ExceptionHandler(ServerException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ResultData<Object> handleServerException(ServerException e) {
        log.error("服务器异常：{}", e.getMessage());
        return ResultData.failure(e.getReturnCodeEnum());
    }
}
```

## 五、异常分类指南

| 异常类型 | 使用场景 | 示例 |
|----------|----------|------|
| `BusinessException` | 业务逻辑异常 | 用户不存在、余额不足、订单已取消 |
| `FileUploadException` | 文件上传相关异常 | 文件过大、格式不支持、上传失败 |
| `ServerException` | 服务器内部异常 | 数据库连接失败、第三方服务调用失败 |
| `AuthException` | 认证授权异常 | Token 过期、权限不足、登录失败 |
| `{Module}Exception` | 模块特定异常 | 根据业务模块自定义 |

## 六、最佳实践

### 1. 异常命名规范

```java
// ✅ 推荐：清晰的异常命名
UserNotFoundException          // 用户未找到
InsufficientBalanceException  // 余额不足
TokenExpiredException         // Token 过期

// ❌ 禁止：模糊的异常命名
UserException                 // 太宽泛
AuthException                 // 不够具体（除非是模块异常基类）
```

### 2. 异常消息规范

```java
// ✅ 推荐：包含具体信息
throw new BusinessException(ReturnCodeEnum.USER_NOT_FOUND, 
    "用户ID: " + userId + " 在系统中不存在");

// ❌ 禁止：过于简单的消息
throw new BusinessException(ReturnCodeEnum.USER_NOT_FOUND, "错误");
```

### 3. 异常层次设计

```
RuntimeException
├── BusinessException          # 业务异常基类
│   ├── UserNotFoundException
│   └── InsufficientBalanceException
├── FileUploadException        # 文件上传异常
├── ServerException            # 服务器异常
└── AuthException              # 认证异常（模块私有）
```

## 七、检查清单

创建异常类前，请确认：

- [ ] 异常类是否继承 `RuntimeException`
- [ ] 是否使用 `@Getter` 注解
- [ ] 是否包含 `ReturnCodeEnum` 字段
- [ ] 字段是否使用 `final` 修饰
- [ ] 是否提供必要的构造函数
- [ ] 是否添加了类注释
- [ ] 是否存放在正确的目录（通用异常或模块私有目录）
- [ ] 类名是否以 `Exception` 结尾
