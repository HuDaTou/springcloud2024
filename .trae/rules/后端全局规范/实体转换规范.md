# 实体转换规范

## 概述

本规范定义了项目中 DTO（数据传输对象）、PO（持久化对象）、VO（视图对象）之间的转换标准，统一使用 `BasecopyProperties` 接口提供的方法进行属性复制。

## 转换关系图

```
┌─────────────────────────────────────────────────────────────────┐
│                        Controller                               │
│  ┌─────────────┐          ┌─────────────┐                      │
│  │  RequestDTO │ ──────►  │  ResponseVO │                      │
│  └─────────────┘          └─────────────┘                      │
└───────────────────────┬─────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│                          Service                                │
│  ┌─────────────┐          ┌─────────────┐          ┌───────────┐│
│  │    DTO      │ ──────►  │     PO      │ ──────►  │    VO     ││
│  │ (入参)      │          │ (数据库实体)│          │ (出参)    ││
│  └─────────────┘          └─────────────┘          └───────────┘│
└─────────────────────────────────────────────────────────────────┘
```

## 核心接口：BasecopyProperties

所有需要转换的类（DTO、PO、VO）必须实现 `BasecopyProperties` 接口：

```java
public interface BasecopyProperties {
    // 创建目标类实例并复制属性
    <V> V copyProperties(Class<V> clazz);
    
    // 创建目标类实例并复制属性，支持额外处理
    <V> V copyProperties(Class<V> clazz, Consumer<V> consumer);
    
    // 从源对象复制属性到当前对象
    <T> T copyFrom(Object sourceObj);
}
```

## 一、PO 转 VO

### 1. 基本转换

```java
// ✅ PO 转单个 VO
User user = userMapper.selectById(id);
UserVO vo = user.copyProperties(UserVO.class);
```

### 2. 列表转换

```java
// ✅ PO 列表转 VO 列表
List<User> users = userMapper.selectList(null);
List<UserVO> vos = users.stream()
        .map(user -> user.copyProperties(UserVO.class))
        .toList();
```

### 3. 复杂转换（含关联数据）

```java
// ✅ 使用 lambda 补充额外字段
return article.copyProperties(ArticleDetailVO.class, vo -> {
    vo.setCategoryName(category.getCategoryName());
    vo.setTags(tags.stream().map(tag -> tag.copyProperties(TagVO.class)).toList());
});
```

## 二、DTO 转 PO

### 1. DTO 定义要求

```java
// ✅ DTO 必须实现 BasecopyProperties 接口
@Data
@Accessors(chain = true)
public class UserCreateDTO implements BasecopyProperties {
    @NotBlank(message = "用户名不能为空")
    private String username;
    
    private String nickname;
    private String email;
}
```

### 2. 基本转换方式

```java
// ✅ 推荐：直接使用 copyProperties
@Override
public ResultData<Void> createUser(UserCreateDTO dto) {
    User user = dto.copyProperties(User.class);
    userMapper.insert(user);
    return ResultData.success();
}
```

### 3. 转换后额外处理

```java
// ✅ 使用 lambda 进行转换后处理
@Override
public ResultData<Void> updateUser(Long id, UserUpdateDTO dto) {
    User user = dto.copyProperties(User.class, u -> {
        u.setId(id);                    // 设置主键
        u.setUpdateTime(LocalDateTime.now()); // 设置更新时间
    });
    userMapper.updateById(user);
    return ResultData.success();
}
```

### 4. 字段类型不一致处理

```java
// ✅ DTO 与 PO 字段类型不一致时的处理
@Override
public ResultData<Void> createMenu(MenuCreateDTO dto) {
    SysMenu menu = dto.copyProperties(SysMenu.class, m -> {
        // DTO 中 parentId 为 String，PO 中为 Long
        m.setParentId(StringUtils.isNotBlank(dto.getParentId()) ? Long.parseLong(dto.getParentId()) : null);
    });
    menuService.save(menu);
    return ResultData.success();
}
```

## 三、转换规则总结

| 转换方向 | 调用方式 | 适用场景 |
|---------|---------|---------|
| PO → VO | `po.copyProperties(VO.class)` | 查询结果返回 |
| PO → VO（复杂） | `po.copyProperties(VO.class, consumer)` | 需要补充额外字段 |
| DTO → PO | `dto.copyProperties(PO.class)` | 创建实体 |
| DTO → PO（带处理） | `dto.copyProperties(PO.class, consumer)` | 更新实体、特殊字段处理 |
| Object → 当前对象 | `this.copyFrom(source)` | 从任意对象复制属性 |

## 四、命名规范

| 对象类型 | 命名后缀 | 说明 |
|---------|---------|------|
| 数据传输对象 | DTO | 用于接口入参 |
| 创建请求对象 | CreateDTO | 专门用于创建操作 |
| 更新请求对象 | UpdateDTO | 专门用于更新操作 |
| 查询条件对象 | QueryDTO/SearchDTO | 用于查询条件 |
| 视图对象 | VO | 用于接口出参 |
| 分页视图对象 | PageVO | 用于分页结果 |
| 持久化对象 | 无后缀 | 数据库实体类 |

## 五、禁止行为

❌ **禁止**：手动逐字段赋值进行转换（代码冗长且易遗漏）
```java
// ❌ 不推荐
User user = new User();
user.setUsername(dto.getUsername());
user.setNickname(dto.getNickname());
```

❌ **禁止**：使用 BeanUtils.copyProperties（性能差且不安全）
```java
// ❌ 不推荐
User user = new User();
BeanUtils.copyProperties(dto, user);
```

✅ **推荐**：统一使用 `copyProperties` 方法
```java
// ✅ 推荐
User user = dto.copyProperties(User.class);
```

## 六、注意事项

1. **字段命名一致性**：DTO、PO、VO 的同名字段必须保持类型兼容
2. **特殊字段处理**：ID、创建时间、更新时间等字段建议在 lambda 中手动设置
