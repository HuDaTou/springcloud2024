# 实体类创建规范

## 一、强制要求（AI必须100%遵守，违反即错误）
1. 必须继承 `BaseData` 类，获得基础字段（createTime、updateTime、deleted）
2. 必须实现 `BasecopyProperties` 接口，支持对象属性拷贝
3. 必须使用 `@Data` 注解（Lombok）简化代码，禁止手动编写 getter/setter 方法
4. 必须使用 `@Accessors(chain = true)` 注解支持链式调用
5. 必须定义无参构造函数，有参构造函数按需添加
6. 必须实现 `Serializable` 接口，确保对象可序列化
7. 必须使用 `@TableName` 注解指定数据库表名，表名使用下划线命名法
8. 主键字段必须使用 `@TableId` 注解，并指定主键生成策略为 `IdType.AUTO`
9. 字段名必须使用驼峰命名法，对应数据库下划线命名
10. 必须使用 `@TableField` 注解明确指定数据库字段名（当字段名与属性名不一致时）
11. 必须使用包装类型（Integer、Long、Boolean等）而非基本类型
12. 必须添加业务字段的枚举约束（如状态字段）
13. 禁止使用 `@Transient` 注解标记数据库字段，应使用 `@TableField(exist = false)`
14. **必须使用 `@Schema` 注解（Swagger）为每个字段添加 API 文档描述**

## 二、推荐做法（AI优先采用，效果更好）
1. 推荐使用枚举类定义状态常量，提高代码可读性
2. 推荐添加字段注释，说明字段含义和业务用途
3. 推荐使用 `@Builder` 注解支持链式调用创建对象
4. 推荐使用静态工厂方法创建对象，封装创建逻辑
5. 推荐对日期字段使用 `LocalDateTime` 类型
6. 推荐使用 `@JsonFormat` 和 `@DateTimeFormat` 注解统一日期格式
7. 推荐添加逻辑删除字段 `deleted`，配合 `@TableLogic` 注解使用
8. **推荐为 `@Schema` 注解提供 `description` 和 `example` 属性**

## 三、禁止行为（AI绝对不能生成）
1. 禁止使用基本数据类型作为实体字段
2. 禁止硬编码数据库表名和字段名
3. 禁止在实体类中编写业务逻辑方法
4. 禁止使用 `Date` 类型，必须使用 `LocalDateTime`
5. 禁止在实体类中定义集合类型字段（应通过关联查询实现）
6. **禁止省略 `@Schema` 注解，每个字段必须有 API 文档描述**

## 四、标准代码示例
```java
package com.example.entity;

import com.baomidou.mybatisplus.annotation.*;
import com.example.enums.StatusEnum;
import com.overthinker.cloud.common.core.base.BaseData;
import com.overthinker.cloud.common.core.base.BasecopyProperties;
import io.swagger.v3.oas.annotations.media.Schema;
import lombok.AllArgsConstructor;
import lombok.Builder;
import lombok.Data;
import lombok.NoArgsConstructor;

/**
 * 用户实体类
 * 对应数据库表：sys_user
 */
@Data
@Accessors(chain = true)
@Builder
@NoArgsConstructor
@AllArgsConstructor
@TableName("sys_user")
@Schema(description = "用户实体")
public class SysUser extends BaseData implements BasecopyProperties {

    @Schema(description = "用户ID，主键，自增", example = "1")
    @TableId(value = "id", type = IdType.AUTO)
    private Long id;

    @Schema(description = "用户名", example = "admin")
    @TableField("username")
    private String username;

    @Schema(description = "用户密码（加密存储）", example = "$2a$10$...")
    @TableField("password")
    private String password;

    @Schema(description = "真实姓名", example = "张三")
    @TableField("real_name")
    private String realName;

    @Schema(description = "手机号码", example = "13800138000")
    @TableField("phone")
    private String phone;

    @Schema(description = "邮箱地址", example = "admin@example.com")
    @TableField("email")
    private String email;

    @Schema(description = "用户状态：0-禁用，1-启用", example = "1")
    @TableField("status")
    private StatusEnum status;

    // 继承 BaseData 的字段：
    // - createTime: 创建时间（自动填充）
    // - updateTime: 更新时间（自动填充）
    // - deleted: 删除标记（逻辑删除）
}
```

```java
package com.example.enums;

import lombok.Getter;

/**
 * 用户状态枚举
 */
@Getter
public enum StatusEnum {
    
    DISABLED(0, "禁用"),
    ENABLED(1, "启用");

    private final Integer code;
    private final String desc;

    StatusEnum(Integer code, String desc) {
        this.code = code;
        this.desc = desc;
    }

    public static StatusEnum fromCode(Integer code) {
        for (StatusEnum status : values()) {
            if (status.getCode().equals(code)) {
                return status;
            }
        }
        throw new IllegalArgumentException("未知状态码: " + code);
    }
}
```

## 五、@Schema 注解使用规范

### 5.1 基本用法
```java
@Schema(description = "字段描述", example = "示例值")
private String fieldName;
```

### 5.2 常用属性
| 属性 | 说明 | 示例 |
|------|------|------|
| `description` | 字段描述（必填） | `"用户昵称"` |
| `example` | 示例值（推荐） | `"张三"` |
| `required` | 是否必填 | `true` |
| `hidden` | 是否隐藏（敏感字段） | `true` |
| `defaultValue` | 默认值 | `"guest"` |

### 5.3 特殊场景

**敏感字段（密码等）应隐藏：**
```java
@Schema(description = "用户密码", hidden = true)
private String password;
```

**日期字段：**
```java
@Schema(description = "创建时间", example = "2024-01-01T12:00:00")
private LocalDateTime createTime;
```

**枚举字段：**
```java
@Schema(description = "用户状态：0-禁用，1-启用", example = "1")
private Integer status;
```
