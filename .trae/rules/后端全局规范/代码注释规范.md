# 代码注释规范

## 一、强制要求（AI必须100%遵守，违反即错误）
1. **必须为所有类添加类注释**：使用Javadoc格式，包含类的功能描述
2. **必须为所有公共方法添加方法注释**：使用Javadoc格式，包含功能说明、参数说明、返回值说明
3. **必须为所有字段添加字段注释**：使用`@Schema`注解或行注释说明字段含义
4. **必须保留现有代码中的注释**：AI生成或修改代码时，不得删除已存在的有效注释
5. **必须为枚举类添加注释**：说明枚举用途和每个枚举值的含义
6. **必须为异常类添加注释**：说明异常的触发条件和使用场景

## 二、推荐做法（AI优先采用，效果更好）
1. 推荐使用Javadoc格式注释，便于生成API文档
2. 推荐为复杂业务逻辑添加详细注释说明
3. 推荐为重要的常量添加注释说明其含义和用途
4. 推荐为条件分支较多的代码添加注释说明业务逻辑
5. 推荐为私有方法也添加适当的注释
6. 推荐使用`@author`、`@since`等Javadoc标签丰富注释内容

## 三、禁止行为（AI绝对不能生成）
1. **禁止删除或修改已存在的有效注释**：包括类注释、方法注释、字段注释和行注释
2. **禁止使用无意义的注释**：如`// TODO`、`// FIXME`等占位符注释（除非有实际待办事项）
3. **禁止注释与代码不一致**：修改代码时必须同步更新相关注释
4. **禁止使用中文标点符号**：注释应使用英文标点
5. **禁止过长的行注释**：单行注释长度不应超过80字符

### 4.2 方法注释规范
```java
/**
 * 根据用户ID查询用户信息
 *
 * @param id 用户ID
 * @return 用户VO对象，不存在返回null
 */
@Override
public UserVO getUserById(Long id) {
    User user = userMapper.selectById(id);
    if (Objects.isNull(user)) {
        return null;
    }
    return user.copyProperties(UserVO.class);
}
```

### 4.3 字段注释规范
```java
/**
 * 用户实体类
 * 对应数据库表：sys_user
 */
@Data
@TableName("sys_user")
public class User extends BaseData {

    /**
     * 用户ID，主键，自增
     */
    @TableId(value = "id", type = IdType.AUTO)
    private Long id;

    /**
     * 用户名（唯一标识）
     */
    @TableField("username")
    private String username;

    /**
     * 用户状态：0-禁用，1-启用
     */
    @TableField("status")
    private Integer status;
}
```

### 4.4 枚举类注释规范
```java
/**
 * 用户状态枚举
 * <p>
 * 定义用户的有效状态值
 * </p>
 */
@Getter
public enum StatusEnum {
    
    /**
     * 禁用状态
     */
    DISABLED(0, "禁用"),
    
    /**
     * 启用状态
     */
    ENABLED(1, "启用");

    private final Integer code;
    private final String desc;

    StatusEnum(Integer code, String desc) {
        this.code = code;
        this.desc = desc;
    }
}
```

### 4.6 复杂逻辑注释规范
```java
/**
 * 处理用户注册业务
 * <p>
 * 流程：校验用户名 -> 校验邮箱 -> 密码加密 -> 保存用户 -> 初始化用户角色
 * </p>
 *
 * @param dto 用户注册信息
 * @return 操作结果
 */
@Override
@Transactional
public ResultData<Void> register(UserRegisterDTO dto) {
    // 1. 校验用户名是否已存在
    if (userMapper.exists(new LambdaQueryWrapper<User>()
            .eq(User::getUsername, dto.getUsername()))) {
        throw new BusinessException(ReturnCodeEnum.USERNAME_EXISTS);
    }

    // 2. 校验邮箱是否已被使用
    if (userMapper.exists(new LambdaQueryWrapper<User>()
            .eq(User::getEmail, dto.getEmail()))) {
        throw new BusinessException(ReturnCodeEnum.EMAIL_EXISTS);
    }

    // 3. 创建用户并加密密码
    User user = User.builder()
            .username(dto.getUsername())
            .password(passwordEncoder.encode(dto.getPassword()))
            .email(dto.getEmail())
            .status(StatusEnum.ENABLED)
            .build();
    
    userMapper.insert(user);

    // 4. 初始化用户角色（默认普通用户角色）
    userRoleMapper.insert(UserRole.builder()
            .userId(user.getId())
            .roleId(RoleEnum.USER.getValue())
            .build());

    return ResultData.success();
}
```

## 五、Javadoc标签使用规范

| 标签 | 说明 | 使用场景 |
|------|------|----------|
| `@param` | 参数说明 | 方法参数描述 |
| `@return` | 返回值说明 | 方法返回值描述 |
| `@throws` | 异常说明 | 方法可能抛出的异常 |
| `@author` | 作者 | 类或方法的作者 |
| `@since` | 版本 | 从哪个版本开始存在 |
| `@see` | 参考链接 | 引用相关类或方法 |
| `@deprecated` | 废弃标记 | 标记不再推荐使用的方法 |

## 六、注释保留规则
1. **类注释**：必须保留，不可删除
2. **方法注释**：必须保留，不可删除
3. **字段注释**：必须保留，不可删除
4. **行注释**：必须保留有业务含义的行注释
5. **块注释**：必须保留有业务含义的块注释
6. **TODO注释**：如果有明确的待办事项可以保留，否则删除
