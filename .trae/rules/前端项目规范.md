# Frontend Project Rules

> **适用范围**：本规范适用于 `frontend/` 目录下所有前端项目。
>
> - ✅ 适用：`frontend/admin/`、`frontend/blog/`
> - ❌ 不适用：`cloud-*/`、`cloud-common/` 等 Java 后端项目

---

## 1. 命名规范

### 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 组件 | 大驼峰或 kebab-case | `UserInfo.vue` / `user-info.vue` |
| TS/JS 文件 | 小驼峰或 kebab-case | `userInfo.ts` / `user-info.ts` |
| 样式文件 | kebab-case | `user-profile.scss` |
| 枚举文件 | 小驼峰 + `Enum` 后缀 | `httpEnum.ts` |
| 工具文件 | 小驼峰或功能名 + `Utils` | `request.ts` / `dateUtils.ts` |
| API 文件 | 小驼峰或按模块目录组织 | `user.ts` / `article/index.ts` |
| Store 文件 | 小驼峰 + `Store` 后缀或按模块 | `user.ts` / `userStore.ts` |

### 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 普通变量 | 小驼峰 | `userName`, `isLoading` |
| 常量 | 全大写 + 下划线 | `MAX_RETRY_COUNT`, `API_BASE_URL` |
| 布尔变量 | `is` / `has` / `can` 前缀 | `isVisible`, `hasPermission` |
| 数组变量 | 复数名词或加 `List` | `users`, `userList` |
| 对象变量 | 名词或加 `Info` / `Data` | `userInfo`, `articleData` |

### CSS 类名命名

使用 BEM 命名规范或 kebab-case：

```scss
// BEM 规范
.user-card {
  &__header { }
  &__body { }
  &--active { }
}

// kebab-case
.user-card { }
.user-card-header { }
```

## 2. TypeScript 规范

### 类型定义

```typescript
// ✅ 推荐：使用 interface 定义对象类型
interface UserInfo {
  id: number
  username: string
  avatar?: string
  roles: string[]
}

// ✅ 推荐：使用 type 定义联合类型或工具类型
type UserStatus = 'active' | 'inactive' | 'banned'
type PartialUser = Partial<UserInfo>

// ✅ 推荐：定义 API 响应类型
interface ApiResponse<T> {
  code: number
  data: T
  msg: string
}

interface PageResult<T> {
  list: T[]
  total: number
  pageNum: number
  pageSize: number
}
```

### 禁止使用

```typescript
// ❌ 禁止：使用 any
function handleData(data: any) { ... }

// ✅ 推荐：使用 unknown 或具体类型
function handleData(data: unknown) { ... }

// ❌ 禁止：使用 var
var name = 'test'

// ✅ 推荐：使用 const / let
const name = 'test'
```

## 3. 组件规范

### 组件结构

```vue
<script setup lang="ts">
// 1. 导入
import { ref, computed, onMounted } from 'vue'
import type { PropType } from 'vue'

// 2. Props 定义
interface Props {
  title: string
  data?: any[]
  onChange?: (val: string) => void
}

const props = withDefaults(defineProps<Props>(), {
  title: '默认标题',
  data: () => []
})

// 3. Emits 定义
const emit = defineEmits<{
  (e: 'update', value: string): void
  (e: 'close'): void
}>()

// 4. 响应式数据
const loading = ref(false)
const localValue = ref('')

// 5. 计算属性
const isValid = computed(() => localValue.value.length > 0)

// 6. 方法
function handleSubmit() {
  emit('update', localValue.value)
}

// 7. 生命周期
onMounted(() => {
  // 初始化
})
</script>

<template>
  <div class="component-name">
    <!-- 模板内容 -->
  </div>
</template>

<style scoped lang="scss">
.component-name {
  // 样式
}
</style>
```

### Props 定义

```typescript
// ✅ 推荐：使用 TypeScript 接口 + withDefaults
interface Props {
  title: string
  count?: number
  list?: string[]
}

const props = withDefaults(defineProps<Props>(), {
  count: 0,
  list: () => []
})
```

### 事件命名

```typescript
// ✅ 推荐：kebab-case 事件名
emit('update:value', newValue)
emit('item-click', item)
```

## 4. 状态管理规范

### Store 定义方式

```typescript
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', () => {
  // 状态
  const userInfo = ref<UserInfo>()
  const token = ref<string>()

  // 计算属性
  const isLoggedIn = computed(() => !!token.value)

  // 方法
  async function fetchUserInfo() {
    const { data } = await getUserInfoApi()
    userInfo.value = data
  }

  function logout() {
    token.value = null
    userInfo.value = undefined
  }

  return {
    userInfo,
    token,
    isLoggedIn,
    fetchUserInfo,
    logout
  }
})
```

### Store 使用规范

```typescript
// ✅ 推荐：组合式函数方式
const userStore = useUserStore()

// 在 setup 或 composable 中使用
function MyComponent() {
  const userStore = useUserStore()
  // ...
}
```

## 5. API 请求规范

### API 封装

```typescript
// utils/request.ts - Axios 封装
export interface ResponseBody<T = any> {
  code: number
  data?: T
  msg: string
}

export function useGet<R = any, T = any>(
  url: string,
  params?: T
): Promise<ResponseBody<R>> {
  return instancePromise({ url, params, method: 'GET' })
}

export function usePost<R = any, T = any>(
  url: string,
  data?: T
): Promise<ResponseBody<R>> {
  return instancePromise({ url, data, method: 'POST' })
}

// api/user/index.ts - API 接口定义
export async function userList() {
  return useGet('/user/list')
}

export async function userDetail(id: string) {
  return useGet(`/user/details/${id}`)
}
```

### API 文件组织

```
// ✅ 推荐：按功能模块组织
apis/
├── user/
│   └── index.ts
├── article/
│   ├── index.ts
│   └── type.ts
```

## 6. 样式规范

### 基本规范

| 规范 | 说明 |
|------|------|
| 使用 scoped | 组件样式添加 `scoped` 避免污染 |
| 避免行内样式 | 优先使用 CSS 类 |
| 使用 CSS 变量 | 主题色等使用变量 |
| 移动优先 | 先写移动端样式 |

### 样式示例

```scss
<style scoped lang="scss">
.user-card {
  padding: 16px;
  background: #fff;

  &__header {
    font-size: 18px;
    font-weight: bold;
  }

  &--active {
    border-color: var(--primary-color);
  }
}
</style>
```

## 7. 路由规范

### 路由规范

| 规范 | 说明 |
|------|------|
| 路由懒加载 | 使用 `() => import()` |
| 路由命名 | 使用 PascalCase 或 kebab-case |
| 路由路径 | 使用 kebab-case |
| 404 处理 | 捕获所有未匹配路由 |

### 路由示例

```typescript
const routes: RouteRecordRaw[] = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/pages/common/login.vue')
  },
  {
    path: '/:pathMatch(.*)*',
    name: 'NotFound',
    component: () => import('@/pages/exception/404.vue')
  }
]
```

## 8. 通用规范

### 技术栈要求

| 要求 | 说明 |
|------|------|
| Vue 3 | 必须使用 Vue 3 |
| Composition API | 必须使用 `<script setup>` |
| Pinia | 状态管理必须使用 Pinia |
| TypeScript | 必须使用 TypeScript，避免 `any` |

### 禁止使用

| 禁止技术 | 原因 | 推荐替代 |
|----------|------|----------|
| Vue 2 | 已停止维护 | Vue 3 |
| Options API | 代码组织性差 | Composition API |
| Vuex | 已停止维护 | Pinia |
| `any` 类型 | 丢失类型安全 | 明确类型定义 |
| `var` | 函数作用域问题 | `const` / `let` |
